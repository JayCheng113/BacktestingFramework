# V2.20.1 Design Draft — NestedOOSStep + Optimizer ABC

**Status:** Draft (for codex round-5 review feedback before implementation)
**Date:** 2026-04-12
**Depends on:** V2.20.0 P1-A MVP (`ez.research` framework)
**Replaces:** `validation/phase_o_nested_oos.py` (411 lines) and similar phase scripts

---

## Goal

Reduce a typical "降回撤研究"-style nested OOS workflow from a 200-400 line one-shot script to a ~30-50 line `ResearchPipeline` declaration. The key abstractions are:

1. **`Optimizer` ABC** — separates "what to optimize" from "how to evaluate"
2. **Multi-objective optimization** — one call returns N candidate weights (one per objective)
3. **`NestedOOSStep`** — IS slice → optimize → OOS slice → validate → record
4. **Built-in objectives library** — `MaxCalmar`, `MaxSharpe`, `EpsilonConstraint`, etc.

This is the first **real-world** test of the `ez.research` framework. If the design is right, V2.20.x can keep adding `WalkForwardStep`, `PairedBootstrapStep`, etc. without refactoring the framework. If it's wrong, we find out now (cheap) instead of after 6 more steps depend on a broken foundation.

---

## Reference: what `phase_o_nested_oos.py` actually does

The 411-line script does five things, each tightly coupled to the others:

1. **Load 3 daily return series** (`ra` = EtfRotateCombo, `re` = bond ETF, `rf` = gold ETF)
2. **8 objectives × 1 IS window** = 8 differential_evolution calls, each producing `(wa, we, wf)` weights
3. **Apply each best weight to OOS window**, compute OOS metrics
4. **Compare to a baseline P1 (A70/E15/F15)**
5. **Print + dump JSON**

The same pattern repeats in `phase_p_walk_forward.py` (rolling re-optimize), `phase_q_paired_bootstrap.py` (paired delta CI), `phase_r_monthly_rebalance.py` (caliber check), and 10+ other scripts. **None of them reuse code** beyond `dynamic_ef_correct.py::calc_metrics`. That's the bug.

```python
# ez/research/optimizers.py (NEW)
from abc import ABC, abstractmethod
from dataclasses import dataclass

import pandas as pd


@dataclass(frozen=True)
class OptimalWeights:
    """Single optimization result.

    Attributes
    ----------
    objective_name : str
        The label of the objective that produced this result.
    weights : dict[str, float]
        {asset_label → weight}, sums to 1.0 (or less if cash position
        allowed). All weights >= 0 (long-only A-share constraint).
    is_metrics : dict[str, float]
        Metrics computed on the IS window for this weight set.
    optimizer_status : str
        "converged" | "infeasible" | "max_iter".
    """
    objective_name: str
    weights: dict[str, float]
    is_metrics: dict[str, float]
    optimizer_status: str


class Objective(ABC):
    """A single optimization objective.

    Subclasses define both the metric to optimize AND any constraints.
    The `evaluate` method takes a portfolio return Series and returns
    a scalar to MINIMIZE (so MaxSharpe returns -sharpe, MinCVaR
    returns +cvar, etc.). `is_feasible` allows epsilon-constraints.
    """
    name: str

    @abstractmethod
    def evaluate(self, port_returns: pd.Series, baseline_metrics: dict | None = None) -> float:
        """Lower is better. Return float('inf') for infeasible solutions."""
        raise NotImplementedError


# Concrete objectives
class MaxSharpe(Objective):
    name = "Max Sharpe"
    def evaluate(self, port_returns, baseline_metrics=None):
        m = compute_basic_metrics(port_returns)
        return -m["sharpe"] if m else float("inf")


class MaxCalmar(Objective):
    name = "Max Calmar"
    def evaluate(self, port_returns, baseline_metrics=None):
        m = compute_basic_metrics(port_returns)
        return -m["calmar"] if m and m["calmar"] > 0 else float("inf")


class MaxSortino(Objective):
    name = "Max Sortino"
    def evaluate(self, port_returns, baseline_metrics=None):
        m = compute_basic_metrics(port_returns)
        return -m.get("sortino", 0) if m else float("inf")


class MinCVaR(Objective):
    def __init__(self, alpha: float = 0.05):
        self.alpha = alpha
        self.name = f"Min CVaR {int(alpha*100)}%"
    def evaluate(self, port_returns, baseline_metrics=None):
        cvar = compute_cvar(port_returns, self.alpha)
        # cvar is negative (loss tail mean); we want to MAXIMIZE it
        # (i.e., minimize the loss). Returning -cvar minimizes the loss.
        return -cvar if cvar is not None else float("inf")


class EpsilonConstraint(Objective):
    """Optimize one metric subject to a constraint on another.

    Examples:
        # Min MDD subject to Ret >= 0.9 * baseline_ret
        EpsilonConstraint(
            objective="min_mdd",
            constraint_metric="ret",
            constraint_op=">=",
            constraint_value="0.9*baseline_ret",
        )

        # Max Ret subject to |MDD| <= 0.6 * baseline_mdd
        EpsilonConstraint(
            objective="max_ret",
            constraint_metric="mdd_abs",
            constraint_op="<=",
            constraint_value="0.6*baseline_mdd",
        )
    """
    OBJECTIVE_FNS = {
        "min_mdd": lambda m: abs(m["dd"]),
        "max_ret": lambda m: -m["ret"],
        "min_vol": lambda m: m["vol"],
    }
    METRIC_KEYS = {"ret", "mdd_abs", "vol", "sharpe", "calmar"}

    def __init__(self, objective, constraint_metric, constraint_op, constraint_value):
        if objective not in self.OBJECTIVE_FNS:
            raise ValueError(f"Unknown objective: {objective}")
        if constraint_metric not in self.METRIC_KEYS:
            raise ValueError(f"Unknown constraint metric: {constraint_metric}")
        if constraint_op not in (">=", "<=", ">", "<"):
            raise ValueError(f"Unsupported op: {constraint_op}")
        self.objective = objective
        self.constraint_metric = constraint_metric
        self.constraint_op = constraint_op
        self.constraint_value = constraint_value
        # Friendly name
        self.name = f"{objective} | {constraint_metric}{constraint_op}{constraint_value}"

    def evaluate(self, port_returns, baseline_metrics=None):
        m = compute_basic_metrics(port_returns)
        if m is None:
            return float("inf")
        # Resolve constraint value (may reference baseline_metrics)
        threshold = self._resolve(self.constraint_value, baseline_metrics)
        actual = self._extract_metric(m, self.constraint_metric)
        if not self._compare(actual, self.constraint_op, threshold):
            return float("inf")  # infeasible
        return self.OBJECTIVE_FNS[self.objective](m)

    def _resolve(self, value, baseline):
        if isinstance(value, (int, float)):
            return float(value)
        if isinstance(value, str):
            # Allow "0.9*baseline_ret" syntax
            if "baseline_" in value:
                if baseline is None:
                    raise ValueError(f"Constraint references baseline but no baseline_metrics provided")
                # SAFE eval: only allow * and / against baseline keys
                # (use ast.parse + walk, not eval)
                return self._safe_eval(value, baseline)
            return float(value)
        raise TypeError(f"Constraint value must be number or str, got {type(value).__name__}")

    def _extract_metric(self, m, key):
        if key == "mdd_abs":
            return abs(m["dd"])
        return m.get(key, 0)

    def _compare(self, actual, op, threshold):
        if op == ">=": return actual >= threshold
        if op == "<=": return actual <= threshold
        if op == ">": return actual > threshold
        if op == "<": return actual < threshold


class Optimizer(ABC):
    """Base class for portfolio weight optimizers."""

    @abstractmethod
    def optimize(
        self,
        returns: pd.DataFrame,
        baseline_metrics: dict | None = None,
    ) -> list[OptimalWeights]:
        """Run optimization on the IS returns.

        Parameters
        ----------
        returns : pd.DataFrame
            Index: dates. Columns: asset labels. Values: daily returns.
            Already sliced to the IS window by the caller — the
            optimizer does NOT do its own date slicing.
        baseline_metrics : dict, optional
            Reference metrics for epsilon-constraint resolution
            (e.g., the alpha sleeve's standalone IS metrics).
        Returns
        -------
        list[OptimalWeights]
            One per objective registered with this optimizer.
        """
        raise NotImplementedError


class SimplexMultiObjectiveOptimizer(Optimizer):
    """Optimize over a probability simplex (weights >= 0, sum <= 1).

    Uses scipy.optimize.differential_evolution. Supports n-asset
    portfolios where n = len(returns.columns). Optimizes over n-1
    weights and computes the n-th as 1 - sum(others), clamped to >= 0.
    """

    def __init__(
        self,
        objectives: list[Objective],
        seed: int = 42,
        max_iter: int = 200,
        pop_size: int = 25,
        tol: float = 1e-7,
    ):
        if not objectives:
            raise ValueError("SimplexMultiObjectiveOptimizer requires at least one objective")
        self.objectives = objectives
        self.seed = seed
        self.max_iter = max_iter
        self.pop_size = pop_size
        self.tol = tol

    def optimize(self, returns, baseline_metrics=None):
        from scipy.optimize import differential_evolution
        n = len(returns.columns)
        if n < 2:
            raise ValueError(f"SimplexMultiObjectiveOptimizer requires >= 2 assets, got {n}")
        labels = list(returns.columns)
        bounds = [(0.0, 1.0)] * (n - 1)

        def _portfolio(w):
            wf = max(1.0 - sum(w), 0.0)
            full_weights = list(w) + [wf]
            port = (returns.values * full_weights).sum(axis=1)
            return pd.Series(port, index=returns.index)

        results = []
        for obj in self.objectives:
            def _fun(w, _obj=obj):
                if any(wi < -1e-9 for wi in w) or sum(w) > 1 + 1e-9:
                    return float("inf")
                port = _portfolio(w)
                return _obj.evaluate(port, baseline_metrics)
            res = differential_evolution(
                _fun, bounds,
                seed=self.seed, maxiter=self.max_iter,
                popsize=self.pop_size, tol=self.tol,
            )
            if res.fun < 1e5:
                w_partial = list(res.x)
                wf = max(1.0 - sum(w_partial), 0.0)
                weights_dict = dict(zip(labels, w_partial + [wf]))
                port = _portfolio(w_partial)
                is_metrics = compute_basic_metrics(port)
                results.append(OptimalWeights(
                    objective_name=obj.name,
                    weights=weights_dict,
                    is_metrics=is_metrics,
                    optimizer_status="converged",
                ))
            else:
                results.append(OptimalWeights(
                    objective_name=obj.name,
                    weights={label: 0.0 for label in labels},
                    is_metrics={},
                    optimizer_status="infeasible",
                ))
        return results
```

### Layer 2 — `ez.research.steps.nested_oos`

```python
# ez/research/steps/nested_oos.py (NEW)
import pandas as pd
from datetime import date

from ..pipeline import ResearchStep
from ..context import PipelineContext
from ..optimizers import Optimizer, OptimalWeights, compute_basic_metrics


class NestedOOSStep(ResearchStep):
    """Nested OOS validation: IS optimize → OOS evaluate → baseline compare.

    Reads:
        artifacts['returns']: pd.DataFrame[date × asset_label] (from RunStrategiesStep)
    Writes:
        artifacts['nested_oos_results']: dict with structure
            {
                "is_window": (str, str),
                "oos_window": (str, str),
                "candidates": [
                    {
                        "objective": str,
                        "weights": dict[str, float],
                        "is_metrics": dict[str, float],
                        "oos_metrics": dict[str, float],
                        "status": str,
                    },
                    ...
                ],
                "baseline_oos": dict | None,
                "baseline_is": dict | None,
            }
    """
    name = "nested_oos"
    writes = ("nested_oos_results",)

    def __init__(
        self,
        is_window: tuple[str | date, str | date],
        oos_window: tuple[str | date, str | date],
        optimizer: Optimizer,
        baseline_weights: dict[str, float] | None = None,
        baseline_label: str | None = None,
    ):
        """
        Parameters
        ----------
        is_window : (start, end)
            In-sample window for optimization. Inclusive.
        oos_window : (start, end)
            Out-of-sample window for validation. Inclusive.
            MUST not overlap with is_window (validated at run time).
        optimizer : Optimizer
            Already-configured optimizer. The step calls
            optimizer.optimize(is_returns, baseline_metrics) and
            collects all candidates.
        baseline_weights : dict[str, float], optional
            Reference weight set for OOS comparison (e.g. P1).
            Keys must match `returns` columns.
        baseline_label : str, optional
            If set, the optimizer's baseline_metrics will be the
            standalone metrics of this single asset on the IS window.
            Used for EpsilonConstraint(value="0.9*baseline_ret").
        """
        if not is_window or len(is_window) != 2:
            raise ValueError("is_window must be (start, end)")
        if not oos_window or len(oos_window) != 2:
            raise ValueError("oos_window must be (start, end)")
        self.is_window = is_window
        self.oos_window = oos_window
        self.optimizer = optimizer
        self.baseline_weights = dict(baseline_weights) if baseline_weights else None
        self.baseline_label = baseline_label

    def _slice(self, returns: pd.DataFrame, window: tuple) -> pd.DataFrame:
        start = pd.Timestamp(window[0])
        end = pd.Timestamp(window[1])
        return returns.loc[(returns.index >= start) & (returns.index <= end)]

    def _validate_no_overlap(self):
        is_end = pd.Timestamp(self.is_window[1])
        oos_start = pd.Timestamp(self.oos_window[0])
        if oos_start <= is_end:
            raise ValueError(
                f"NestedOOSStep: OOS window starts {oos_start.date()} which is "
                f"<= IS end {is_end.date()}. OOS must be strictly after IS."
            )

    def run(self, context: PipelineContext) -> PipelineContext:
        self._validate_no_overlap()
        returns = context.require("returns")

        # IS slice
        is_returns = self._slice(returns, self.is_window)
        if is_returns.empty:
            raise RuntimeError(
                f"NestedOOSStep: IS window {self.is_window} produced empty slice. "
                f"Returns date range: {returns.index.min().date()} to {returns.index.max().date()}"
            )
        oos_returns = self._slice(returns, self.oos_window)
        if oos_returns.empty:
            raise RuntimeError(
                f"NestedOOSStep: OOS window {self.oos_window} produced empty slice."
            )

        # Compute baseline metrics on IS for epsilon-constraint resolution
        baseline_metrics = None
        if self.baseline_label:
            if self.baseline_label not in is_returns.columns:
                raise ValueError(
                    f"baseline_label '{self.baseline_label}' not in returns columns: "
                    f"{list(is_returns.columns)}"
                )
            baseline_metrics = compute_basic_metrics(is_returns[self.baseline_label])

        # IS optimize → list of candidates
        candidates = self.optimizer.optimize(is_returns, baseline_metrics)

        # OOS validate each candidate
        validated = []
        for cand in candidates:
            port_full = sum(returns[label] * w for label, w in cand.weights.items())
            port_oos = self._slice(port_full.to_frame("port"), self.oos_window)["port"]
            oos_metrics = compute_basic_metrics(port_oos)
            validated.append({
                "objective": cand.objective_name,
                "weights": cand.weights,
                "is_metrics": cand.is_metrics,
                "oos_metrics": oos_metrics or {},
                "status": cand.optimizer_status,
            })

        # Baseline OOS reference
        baseline_oos_metrics = None
        baseline_is_metrics_for_report = None
        if self.baseline_weights:
            missing = set(self.baseline_weights) - set(returns.columns)
            if missing:
                raise ValueError(
                    f"baseline_weights references unknown labels: {missing}. "
                    f"Available: {list(returns.columns)}"
                )
            port_full = sum(returns[label] * w for label, w in self.baseline_weights.items())
            port_is = self._slice(port_full.to_frame("port"), self.is_window)["port"]
            port_oos = self._slice(port_full.to_frame("port"), self.oos_window)["port"]
            baseline_is_metrics_for_report = compute_basic_metrics(port_is)
            baseline_oos_metrics = compute_basic_metrics(port_oos)

        context.artifacts["nested_oos_results"] = {
            "is_window": (str(self.is_window[0]), str(self.is_window[1])),
            "oos_window": (str(self.oos_window[0]), str(self.oos_window[1])),
            "candidates": validated,
            "baseline_is": baseline_is_metrics_for_report,
            "baseline_oos": baseline_oos_metrics,
        }
        return context
```

## Open questions

### Q1 — Symbol vs label dichotomy

`RunStrategiesStep` currently requires `dict_key == universe_data symbol`. So if I use labels `"A"/"E"/"F"` in NestedOOSStep, I'd need to either:

(a) Pre-create universe_data with `"A"/"E"/"F"` as keys (DataLoadStep doesn't support this yet — symbols are the data fetch keys)
(b) Use real symbols in `strategies={...}` then map back: `baseline_weights={"511010.SH": 0.15, "518880.SH": 0.15, "EtfRotateCombo_X": 0.70}` — ugly
(c) Add a `label_map` parameter to `RunStrategiesStep`: `RunStrategiesStep(strategies={"A": EtfRotateCombo()}, symbol_map={"A": "EtfRotateCombo_synthetic"})`

Option (c) is the cleanest. **This is V2.20.1 prerequisite work** — should be fixed before NestedOOSStep is implemented, or the user-facing API is awkward.

### Q2 — How to fetch the alpha sleeve's daily returns?

Phase O computes `ra = run_strat(EtfRotateCombo(), ...)` which is a portfolio strategy daily-return series, NOT a single ETF's returns. `DataLoadStep` only does `fetch_kline_df` on individual symbols.

Two options:

(a) `RunStrategiesStep` already handles this — it runs `EtfRotateCombo()` on a universe of ETFs and produces a single daily return Series in `artifacts['returns']['A']`. So the dict_key "A" maps to a virtual asset whose data is the strategy's output. **This works** as long as Q1 is solved (label_map).

(b) Add a `RunPortfolioStep` that takes portfolio strategies (multi-symbol) and produces a single equity/return curve in `artifacts['returns']`.

Option (a) is the answer — `RunStrategiesStep` is already designed to handle this case. The output `returns[label]` is what the strategy produced, regardless of whether it ran on 1 symbol or 30.

But there's a problem: `RunStrategiesStep` uses `VectorizedBacktestEngine` which is **single-symbol**. `EtfRotateCombo` is a portfolio strategy that needs a universe. So `RunStrategiesStep` doesn't actually support EtfRotateCombo today.

**This is the architectural issue codex P3-2 flagged**: `RunStrategiesStep` is bound to single-symbol engine. To do nested OOS on the alpha sleeve, we need either:

(a) `RunPortfolioStep` for portfolio strategies (V2.20.2 scope?)
(b) Refactor `RunStrategiesStep` to dispatch single vs portfolio based on strategy type (cleaner)

**Decision**: V2.20.1 should add `RunPortfolioStep` alongside NestedOOSStep. Without it, the most compelling use case (`EtfRotateCombo` as the alpha sleeve) doesn't work.

### Q3 — `compute_basic_metrics` location

The Optimizer ABC and NestedOOSStep both need `compute_basic_metrics` (Sharpe / Calmar / Sortino / MDD / Ret / Vol / CVaR). This function exists in 3 places:
- `validation/dynamic_ef_correct.py::calc_metrics` (the validation/ legacy version)
- `ez/backtest/metrics.py::MetricsCalculator` (the production version, returns slightly different shape)
- `ez/portfolio/metrics.py::resample` (different concern, just resampling)

V2.20.1 should NOT add a 4th implementation. Two reasonable options:

(a) `ez/research/optimizers.py` re-uses `ez/backtest/metrics.py::MetricsCalculator` — most consistent with V2.12.2 metric standardization.
(b) Add a thin wrapper `ez/research/_metrics.py::compute_basic_metrics(returns)` that delegates to `MetricsCalculator` and renames keys (`sharpe_ratio → sharpe`, etc.) for the optimizer's convenience.

**Decision**: option (b). The optimizer needs short keys (`sharpe`, `calmar`, `mdd_abs`, `ret`) that don't match `MetricsCalculator`'s long form (`sharpe_ratio`, `total_return`, `max_drawdown`). A thin wrapper keeps consistency with V2.12.2 formulas while providing the optimizer-friendly interface.

### Q4 — `EpsilonConstraint` safe-eval for `"0.9*baseline_ret"`

The string DSL is convenient but introduces an eval-like surface. V2.20.1 must implement `_safe_eval` using `ast.parse` + walk, NOT Python `eval()`. Allowed operations:
- Multiplication / division by floats
- Reference to `baseline_<key>` where key is one of `MetricsCalculator`'s keys

Anything else (function calls, attribute access, subscripts) → `ValueError`.

Alternative: **drop the string DSL entirely** and force the user to use a callable:

```python
EpsilonConstraint("min_mdd", "ret", ">=", lambda baseline: 0.9 * baseline["ret"])
```

This is cleaner but uglier in the common case. **Decision**: support both (string DSL as syntactic sugar, callable as the underlying implementation).

### Q5 — Test strategy for V2.20.1

The optimizer is non-trivial (scipy.differential_evolution is non-deterministic without a seed). Tests need:

1. **`Optimizer` ABC contract test**: a stub optimizer that returns fixed weights, verify NestedOOSStep wires inputs/outputs correctly.
2. **`SimplexMultiObjectiveOptimizer` golden test**: small synthetic 3-asset returns DataFrame with known optimal weights for `MaxSharpe` (e.g., known equal-weight is optimal for symmetric returns), verify the optimizer recovers them within tolerance.
3. **`EpsilonConstraint` test**: stub returns + a constraint that should reject obvious dominators, verify the optimizer returns "infeasible".
4. **`NestedOOSStep` integration**: end-to-end with synthetic returns, verify IS / OOS slicing is correct, baseline_metrics are computed only on IS, and OOS validation uses untouched OOS data.
5. **NestedOOSStep + ReportStep e2e**: verify the report contains the candidates table.

Estimated: ~30-40 new tests.

### Q6 — Should NestedOOSStep also support `Walk-Forward` mode?

`phase_p_walk_forward.py` is a different step (rolling re-optimize across years). I think NO — make it a separate `WalkForwardStep` in V2.20.2. Each step should do one thing.

### Q7 — Backwards compatibility for `validation/` scripts

The legacy `phase_o_nested_oos.py` script should continue to work after V2.20.1. We're not deleting it. The new `ez.research`-based version becomes the recommended path; the legacy script becomes a "see how it was done before" reference.
