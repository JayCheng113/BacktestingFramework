# V2.13 ML Alpha + 多策略组合 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a walk-forward ML factor framework (`MLAlpha` extending `CrossSectionalFactor`) with anti-lookahead enforcement + purge + embargo, a multi-strategy ensemble combiner (`StrategyEnsemble` extending `PortfolioStrategy`), ML diagnostics (`MLDiagnostics`), user-writable `ml_alphas/` directory with sandboxed contract testing, API endpoints, and a frontend code editor + configuration + diagnostics panel. All additions live in `ez/portfolio/` and `ez/api/routes/portfolio.py`. **Zero CORE file changes.**

**Architecture:** MLAlpha is a stateful `CrossSectionalFactor` subclass that holds an internal trained model and retrains lazily when the eval date moves past its validity window. Model persistence is in-memory only for V1 (fold-local cache; `copy.deepcopy()` per fold isolates state). Purge and embargo are enforced inside `MLAlpha._build_training_panel()` before `model_factory()` is called. StrategyEnsemble wraps a list of inner `PortfolioStrategy` instances, calls each sub-strategy's `generate_weights()`, and combines outputs (equal / IC-weighted / risk-parity) before returning to the engine. MLDiagnostics operates on a completed MLAlpha instance + universe_data, walking through retrain checkpoints and reporting feature-importance stability + IS/OOS IC decay + turnover.

**Tech Stack:** `scikit-learn>=1.3` (new dep, added under `[project.optional-dependencies].ml`), existing pandas / numpy / scipy / FastAPI / React / ECharts. LightGBM / XGBoost are **deferred to V2.13.1** (sklearn-only in V1 keeps pickling + n_jobs edge cases manageable).

## First-Principles Rationale

Every design decision in this plan is derived from one of four fundamental constraints. When implementing, if you find yourself deviating, re-read this section and check whether the deviation is justified.

### Constraint 1 — Users cannot be trusted to enforce walk-forward

Given a Python function `strategy.generate_signals(data: DataFrame)`, a user who trains an ML model inside `generate_signals` will see the entire DataFrame and fit to future returns. The backtest Sharpe will be meaningless. This is not hypothetical — it is the single most common way ML backtests lie.

**Therefore**: MLAlpha cannot be a function the user writes. It must be a framework component that receives:
- A pure `model_factory` (returns a fresh, unfit estimator)
- A pure `feature_fn` (maps a DataFrame slice to features)
- A pure `target_fn` (maps a DataFrame slice to labels)

The framework controls train/predict split. The user never touches the split. This is the core anti-cheat mechanism.

### Constraint 2 — Anti-lookahead means STRICTLY less-than

V2.12.2 round 1 verified `slice_universe_data` uses `df.index.date < target_date` (strict `<`, not `<=`). MLAlpha inherits this guarantee via `CrossSectionalFactor.compute(sliced, dt)`. BUT this is only the framework slice. MLAlpha's training data construction adds a second requirement: if the prediction target is `returns[t : t + k]`, then any training sample whose feature date is `s` must satisfy `s + k <= t - embargo`, otherwise the training label contains information about the prediction window.

**Therefore**: `purge_days` must be at least `target_horizon` of the label. `embargo_days` adds further safety margin for autocorrelation. V2.13 enforces this at `_build_training_panel()` by slicing the training panel with `training_date <= prediction_date - purge_days - embargo_days`.

### Constraint 3 — sandbox `pickle` / `multiprocessing` / `threading` bans force in-process single-thread design

`ez/agent/sandbox.py:_FORBIDDEN_MODULES` blocks `pickle`, `multiprocessing`, `threading`, `subprocess`. This means:
- User code CANNOT persist models to disk via `joblib.dump` (joblib uses pickle under the hood) or via `pickle.dump`
- User code CANNOT use `sklearn.RandomForestRegressor(n_jobs=-1)` (triggers multiprocessing import at fit time)
- User code CANNOT spawn GPU training subprocesses
- User code CANNOT use native `threading.Thread`

**BUT**:
- `copy.deepcopy()` from inside framework code (ez/backtest or ez/portfolio) is NOT sandbox-restricted (sandbox only constrains user files under `strategies/`, `factors/`, `portfolio_strategies/`, `cross_factors/`, `ml_alphas/`)
- BLAS parallelism via numpy / scipy / sklearn native C threads is NOT blocked (it bypasses Python `threading`)
- Single-thread sklearn estimators work without issue (`n_jobs=1` default)

**Therefore**:
1. **V1 model persistence** = in-memory only. Each MLAlpha instance holds its current model in `self._current_model`. `portfolio_walk_forward` deepcopies the MLAlpha instance per fold (via `strategy_factory()`), giving each fold its own independent model state. No disk I/O from user code. No pickle from user code.
2. **n_jobs=1 enforcement**: user template prominently sets `n_jobs=1` on all estimators. Contract test refuses code that passes `n_jobs=-1`. Sandbox AST check is extended to detect the pattern.
3. **GPU training is out of V2.13 scope**. Document as "V2.14+ with proper sandbox escape hatch".

### Constraint 4 — Multi-strategy diversification is additive, not multiplicative

A single ML alpha overfits because it optimizes one objective on one dataset. Combining multiple weakly-correlated alphas reduces idiosyncratic variance by `√N`. Ensemble is not a separate product — it is the natural expression of "I do not trust any single model".

**Therefore**: `StrategyEnsemble` is a `PortfolioStrategy` subclass (not a separate engine mode). The existing engine sees one strategy (the ensemble); internally the ensemble orchestrates sub-strategies. No engine modification. This preserves the "single strategy param" invariant and lets ensemble composition be arbitrarily nested.

```python
"""V2.13 F8: MLAlpha — walk-forward ML factor framework.

MLAlpha is a stateful CrossSectionalFactor that holds a trained sklearn-
compatible model internally and retrains lazily when the current eval
date moves beyond the model's validity window.

Anti-lookahead is enforced at two layers:
1. The framework calls `compute(sliced_data, dt)` where `sliced_data`
   already excludes dates >= dt (via slice_universe_data).
2. `_build_training_panel()` further excludes training samples whose
   forward-looking labels would leak into the prediction window via
   purge_days + embargo_days margin.

Model persistence is in-memory only (V1). Cross-fold sharing is NOT
supported; each walk-forward fold gets a fresh MLAlpha instance via
strategy_factory → copy.deepcopy path, so models are retrained per fold.

See docs/superpowers/plans/2026-04-06-v213-ml-alpha.md for design rationale.
"""
from __future__ import annotations

from datetime import date, datetime, timedelta
from typing import Callable, Any

import numpy as np
import pandas as pd

from ez.portfolio.cross_factor import CrossSectionalFactor


FeatureFn = Callable[[pd.DataFrame], pd.DataFrame]
TargetFn = Callable[[pd.DataFrame], pd.Series]
ModelFactory = Callable[[], Any]


class MLAlpha(CrossSectionalFactor):
    """Walk-forward ML factor. See module docstring for design.

    Args:
        name: Unique factor name (e.g., "ridge_momentum_v1").
        model_factory: Callable returning a fresh unfit estimator.
            MUST return a NEW instance each call (framework will call
            it multiple times across retrain boundaries).
        feature_fn: Per-symbol feature extractor. Called with a
            single-symbol DataFrame (sliced to training window),
            returns a DataFrame of features indexed by date.
        target_fn: Per-symbol target extractor. Called with the same
            single-symbol DataFrame, returns a Series of forward-
            looking labels aligned to the feature dates.
        train_window: Number of trailing trading days to use for each
            retrain's training panel.
        retrain_freq: Retrain when current prediction date exceeds last
            retrain date by this many calendar days.
        purge_days: Exclude the last N training dates before the
            prediction date to prevent feature-label temporal overlap.
            MUST be at least the target's forward horizon.
        embargo_days: Additional buffer on top of purge_days. Defaults
            to 0 (purge is already the minimum safe gap).
    """

    def __init__(
        self,
        name: str,
        model_factory: ModelFactory,
        feature_fn: FeatureFn,
        target_fn: TargetFn,
        train_window: int,
        retrain_freq: int,
        purge_days: int,
        embargo_days: int = 0,
    ):
        if train_window <= 0:
            raise ValueError(f"train_window must be > 0, got {train_window}")
        if retrain_freq <= 0:
            raise ValueError(f"retrain_freq must be > 0, got {retrain_freq}")
        if purge_days < 0:
            raise ValueError(f"purge_days must be >= 0, got {purge_days}")
        if embargo_days < 0:
            raise ValueError(f"embargo_days must be >= 0, got {embargo_days}")

        self._name = name
        self._model_factory = model_factory
        self._feature_fn = feature_fn
        self._target_fn = target_fn
        self._train_window = train_window
        self._retrain_freq = retrain_freq
        self._purge_days = purge_days
        self._embargo_days = embargo_days

        # Runtime state — reset by copy.deepcopy implicitly per fold
        self._current_model: Any = None
        self._last_retrain_date: date | None = None
        self._retrain_count: int = 0

    @property
    def name(self) -> str:
        return self._name

    @property
    def warmup_period(self) -> int:
        # Need at least train_window + purge + embargo days of history
        # before the first prediction date can be made.
        return self._train_window + self._purge_days + self._embargo_days

    def compute(self, universe_data: dict[str, pd.DataFrame], dt: datetime) -> pd.Series:
        """Return per-symbol predictions at date dt."""
        # To be implemented in task 1.4
        return pd.Series(dtype=float)

    def compute_raw(self, universe_data: dict[str, pd.DataFrame], dt: datetime) -> pd.Series:
        """Raw predictions (no ranking). Used by AlphaCombiner."""
        return self.compute(universe_data, dt)
```

```python
class TestMLAlphaLazyRetrain:
    """MLAlpha must call _retrain on first compute call."""

    def _make_universe(self, n_days=200, n_stocks=5, seed=42):
        rng = np.random.default_rng(seed)
        dates = pd.date_range("2022-01-03", periods=n_days, freq="B")
        data = {}
        for i in range(n_stocks):
            prices = 100 * np.cumprod(1 + rng.normal(0.0005, 0.01, n_days))
            data[f"S{i:02d}"] = pd.DataFrame({
                "open": prices, "high": prices * 1.01, "low": prices * 0.99,
                "close": prices, "adj_close": prices,
                "volume": rng.integers(100_000, 1_000_000, n_days).astype(float),
            }, index=dates)
        return data, dates

    def test_compute_triggers_retrain_on_first_call(self):
        from ez.portfolio.ml_alpha import MLAlpha
        from sklearn.linear_model import Ridge

        data, dates = self._make_universe(n_days=200)

        alpha = MLAlpha(
            name="t",
            model_factory=lambda: Ridge(alpha=1.0),
            feature_fn=lambda df: pd.DataFrame({
                "ret1": df["adj_close"].pct_change(1),
                "ret5": df["adj_close"].pct_change(5),

        assert alpha._current_model is None
```

```python
    def _build_training_panel(
        self,
        universe_data: dict[str, pd.DataFrame],
        prediction_date: date,
    ) -> tuple[pd.DataFrame | None, pd.Series | None]:
        """Stack per-symbol (feature, target) pairs into a cross-sectional
        training panel.

        Exclusion rules:
        1. Exclude all dates >= prediction_date - purge_days - embargo_days
           (forward-looking label leakage prevention)
        2. Include at most the last `train_window` dates before the cutoff
        """
        cutoff = prediction_date - timedelta(days=self._purge_days + self._embargo_days)

        rows: list[pd.DataFrame] = []
        labels: list[pd.Series] = []
        for sym, df in universe_data.items():
            if not isinstance(df.index, pd.DatetimeIndex):
                continue
            # Extract features and target for this symbol
            try:
                sym_features = self._feature_fn(df)
                sym_target = self._target_fn(df)
            except Exception:
                continue
            if sym_features is None or sym_target is None:
                continue
            if sym_features.empty or sym_target.empty:
                continue

            # Align features and target on date index
            aligned_idx = sym_features.index.intersection(sym_target.index)
            if len(aligned_idx) == 0:
                continue
            feat_aligned = sym_features.loc[aligned_idx]
            tgt_aligned = sym_target.loc[aligned_idx]

            # Apply purge + embargo: exclude dates >= cutoff
            mask = feat_aligned.index.date < cutoff
            feat_aligned = feat_aligned.loc[mask]
            tgt_aligned = tgt_aligned.loc[mask]

            # Drop rows where target is NaN (pct_change+shift produces NaN tail)
            valid = ~tgt_aligned.isna()
            feat_aligned = feat_aligned.loc[valid]
            tgt_aligned = tgt_aligned.loc[valid]

            if len(feat_aligned) == 0:
                continue

            # Take last train_window dates only
            if len(feat_aligned) > self._train_window:
                feat_aligned = feat_aligned.iloc[-self._train_window:]
                tgt_aligned = tgt_aligned.iloc[-self._train_window:]

            # Add symbol as a MultiIndex level for panel assembly
            feat_aligned = feat_aligned.copy()
            feat_aligned.index = pd.MultiIndex.from_arrays(
                [feat_aligned.index, [sym] * len(feat_aligned)],
                names=["date", "symbol"],
            )
            tgt_aligned.index = feat_aligned.index

            rows.append(feat_aligned)
            labels.append(tgt_aligned)

        if not rows:
            return None, None

        X = pd.concat(rows, axis=0)
        y = pd.concat(labels, axis=0)
        # Drop any remaining NaN rows in X
        valid_rows = ~X.isna().any(axis=1)
        X = X.loc[valid_rows]
        y = y.loc[valid_rows]

        if len(X) == 0:
            return None, None
        return X, y
```

```python
            except Exception:
                continue
```

```python
"""V2.13 F9: MLDiagnostics — overfitting detection for MLAlpha instances.

Diagnostics computed:
1. Feature importance stability: walk through retrains, collect
   model.coef_ or model.feature_importances_, compute std/mean
   (coefficient of variation) per feature. High CV = unstable model.
2. IS/OOS IC decay: at each retrain, compute IC on the training
   window (IS) and on the holdout window immediately after (OOS).
   Large IS-OOS gap = overfitting.
3. Turnover analysis: per-rebalance change in top-N holdings,
   mean absolute position change.
4. Retrain cadence: gaps between retrain events, matches expected
   retrain_freq.

MLDiagnostics operates on a TRAINED MLAlpha instance. It does NOT
retrain from scratch; it re-runs compute() over a diagnostic date
range to capture the snapshot state at each retrain boundary.
"""
from __future__ import annotations

from dataclasses import dataclass, field
from datetime import date, datetime
from typing import Any

import numpy as np
import pandas as pd

from ez.portfolio.ml_alpha import MLAlpha


@dataclass
class DiagnosticsResult:
    feature_importance_cv: dict[str, float] = field(default_factory=dict)
    """Coefficient of variation per feature. Lower = more stable."""

    ic_series: list[tuple[str, float, float]] = field(default_factory=list)
    """List of (date, is_ic, oos_ic). IS = in-sample IC; OOS = next-window IC."""

    retrain_dates: list[str] = field(default_factory=list)
    """ISO date strings of retrain events."""

    avg_turnover: float = 0.0
    """Mean absolute change in top-N holdings per rebalance."""

    overfitting_score: float = 0.0
    """(IS_IC - OOS_IC) / max(|IS_IC|, 1e-6). >0.5 = severe overfitting."""


class MLDiagnostics:
    """Diagnose overfitting risk of a trained MLAlpha."""

    def __init__(self, alpha: MLAlpha):
        self._alpha = alpha

    def run(
        self,
```

```python
"""V2.13 D5: StrategyEnsemble — multi-strategy long-only combiner.

Wraps a list of PortfolioStrategy instances and combines their outputs
using equal / IC-weighted / risk-parity weighting. Output is a single
dict[symbol → weight] compatible with the existing engine's optimizer
and allocator layers.
"""
from __future__ import annotations

from datetime import datetime
from typing import Literal

import numpy as np
import pandas as pd

from ez.portfolio.portfolio_strategy import PortfolioStrategy


EnsembleMode = Literal["equal", "ic_weighted", "risk_parity"]


class StrategyEnsemble(PortfolioStrategy):
    """Combine multiple sub-strategies into a single weight vector.

    Args:
        strategies: List of sub-strategies. Each must implement
            `generate_weights()`. They are called in order at each
            rebalance and their outputs are combined.
        mode: Combination mode.
            - "equal": arithmetic mean weights
            - "ic_weighted": requires IC history (Phase 3 Task 3.3)
            - "risk_parity": requires return covariance (Phase 3 Task 3.4)
        ensemble_weights: Override for manual weights per sub-strategy.
            If provided, overrides the mode's computed weights.
    """

    def __init__(
        self,
        strategies: list[PortfolioStrategy],
        mode: EnsembleMode = "equal",
        ensemble_weights: list[float] | None = None,
    ):
        super().__init__()
        if not strategies:
            raise ValueError("StrategyEnsemble requires at least one strategy")
        if mode not in ("equal", "ic_weighted", "risk_parity"):
            raise ValueError(f"Invalid mode: {mode}")
        if ensemble_weights is not None and len(ensemble_weights) != len(strategies):
            raise ValueError(
                f"ensemble_weights length {len(ensemble_weights)} does not "
                f"match strategies length {len(strategies)}"
            )
        self._strategies = strategies
        self._mode = mode
        self._manual_weights = ensemble_weights

    @property
    def lookback_days(self) -> int:
        return max(s.lookback_days for s in self._strategies)

    def generate_weights(
        self,
        universe_data,
        date,
        prev_weights,
        prev_returns,
    ) -> dict[str, float]:
        # Collect sub-strategy outputs
        sub_outputs: list[dict[str, float]] = []
        for s in self._strategies:
            try:
                out = s.generate_weights(universe_data, date, prev_weights, prev_returns)
                sub_outputs.append(out or {})
            except Exception:
                sub_outputs.append({})

        # Compute ensemble weights
        if self._manual_weights is not None:
            ew = list(self._manual_weights)
        elif self._mode == "equal":
            n = len(self._strategies)
            ew = [1.0 / n] * n
        else:
            # ic_weighted + risk_parity implemented in later tasks
            n = len(self._strategies)
            ew = [1.0 / n] * n

        # Weighted combination
        combined: dict[str, float] = {}
        for sub_out, w in zip(sub_outputs, ew):
            for sym, weight in sub_out.items():
                combined[sym] = combined.get(sym, 0.0) + w * weight

        return combined
```

```python
def _check_n_jobs_safe(tree: ast.AST) -> list[str]:
    """Scan for n_jobs keyword arguments with values != 1.

    sklearn estimators accept n_jobs=-1 to use multiprocessing, which
    is blocked by the sandbox. Warn the user at sandbox-check time
    rather than letting them discover at runtime.
    """
    warnings = []
    for node in ast.walk(tree):
        if isinstance(node, ast.Call):
            for kw in node.keywords:
                if kw.arg == "n_jobs":
                    if isinstance(kw.value, ast.Constant):
                        if kw.value.value not in (1, None):
                            warnings.append(
                                f"Line {node.lineno}: n_jobs={kw.value.value} "
                                f"will fail at runtime (sandbox blocks "
                                f"`multiprocessing`). Set n_jobs=1."
                            )
                    elif isinstance(kw.value, ast.UnaryOp):
                        # n_jobs=-1 case
                        if isinstance(kw.value.operand, ast.Constant):
                            warnings.append(
                                f"Line {node.lineno}: n_jobs=-{kw.value.operand.value} "
                                f"will fail at runtime."
                            )
    return warnings
```
