# V2.13.1 Phase 5: API Endpoints for MLAlpha / MLDiagnostics

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Connect V2.13's Python API layer (MLAlpha / MLDiagnostics) to the REST API. Close the startup-scan gap AND the code.py template/files gap so `ml_alpha` is a fully first-class kind at the API layer.

**Architecture:** Four changes:
1. `loader.py` adds `load_ml_alphas()` — startup scan of `ml_alphas/` directory
2. `code.py` updates `/refresh` (clean + reload), `/template` valid_kinds, `/files` routing
3. `portfolio.py` adds `POST /ml-alpha/diagnostics` — new endpoint
4. Integration test verifies `/evaluate-factors` works with MLAlpha (no endpoint code change)

---

## Root Cause Analysis

Phase 4 wired `ml_alpha` into the **sandbox** (save/validate/hot-reload via `_KIND_DIR_MAP` + `_VALID_KINDS`). But two gaps remain:

1. **Startup-scan gap**: `loader.py` doesn't scan `ml_alphas/`. After server restart, user MLAlpha subclasses vanish from the registry.
2. **API routing gap**: `code.py` has hardcoded `valid_kinds` tuples in `/template` and `/files` that don't include `"ml_alpha"`. Users can't list or create ml_alpha files through the standard code API.

Both are the same class of bug: `_KIND_DIR_MAP` was extended (Phase 4) but downstream consumers of kind lists weren't updated.

---

## Design Decisions (codex round 1 confirmed)

1. **`/refresh` must clean AND reload**: Not just call `load_ml_alphas()` — must first clear `ml_alphas.*` from `sys.modules` + CrossSectionalFactor dual-dict registry, same pattern as the 4 existing kinds (lines 308-319 of `code.py`).

2. **Tests must not depend on real data providers**: Loader test uses `tmp_path` + monkeypatch `_user_root()`. Diagnostics endpoint test mocks `_fetch_data` with synthetic universe data. No `pytest.skip` on 502 — all happy paths must be deterministically testable.

3. **`code.py` template/files must include `ml_alpha`**: `/template` valid_kinds tuple adds `"ml_alpha"`. `/files` routing adds `"ml_alpha"` → `list_portfolio_files("ml_alpha")`.

4. **Identifier contract**: `ml_alpha_name` accepts anything `CrossSectionalFactor.resolve_class()` accepts — class name (`"MyRidgeAlpha"`) or module.class key (`"ml_alphas.ridge.MyRidgeAlpha"`). Documented in request model.

5. **Missing sklearn → 422 (not 500)**: Endpoint wraps MLAlpha instantiation in try/except. If sklearn is missing, `ImportError` is caught and returned as 422 with `"scikit-learn>=1.5 is required for MLAlpha"`.

6. **`_fetch_data` signature**: Uses existing `_fetch_data(symbols, market, start, end, lookback_days=...)` — pass `alpha.warmup_period + 50` as `lookback_days`. No new helper needed.

### Task 5.2: `POST /api/portfolio/ml-alpha/diagnostics` endpoint

**Files:** `ez/api/routes/portfolio.py`, `tests/test_api/test_ml_alpha_api.py`

**Request model:**
```python
class MLDiagnosticsRequest(BaseModel):
    ml_alpha_name: str = Field(
        ...,
        description="Class name or module.class key of the MLAlpha subclass. "
                    "Resolved via CrossSectionalFactor.resolve_class().",
    )
    symbols: list[str] = Field(..., min_length=1)
    market: str = Field(default="cn_stock", pattern=r"^(cn_stock|us_stock|hk_stock)$")
    start_date: date | None = None
    end_date: date | None = None
    eval_freq: str = Field(default="weekly", pattern=r"^(daily|weekly|monthly|quarterly)$")
    forward_horizon: int = Field(default=5, ge=1, le=60)
    severe_overfit_threshold: float = Field(default=0.5, gt=0, le=2.0)
    mild_overfit_threshold: float = Field(default=0.2, gt=0, le=2.0)
    high_turnover_threshold: float = Field(default=0.6, gt=0, le=1.0)
    top_n_for_turnover: int = Field(default=10, ge=1, le=100)
```

**Endpoint:**
```python
@router.post("/ml-alpha/diagnostics")
def ml_alpha_diagnostics(req: MLDiagnosticsRequest):
    # Lazy imports — sklearn is optional, avoid circular deps
    from ez.portfolio.cross_factor import CrossSectionalFactor
    try:
        from ez.portfolio.ml_alpha import MLAlpha
    except ImportError as e:
        raise HTTPException(422, f"scikit-learn>=1.5 required: {e}")
    from ez.portfolio.ml_diagnostics import MLDiagnostics, DiagnosticsConfig

    # 1. Resolve class (KeyError → 404, ValueError → 404)
    try:
        cls = CrossSectionalFactor.resolve_class(req.ml_alpha_name)
    except (KeyError, ValueError) as e:
        raise HTTPException(404, f"MLAlpha '{req.ml_alpha_name}' not found: {e}")

    if not issubclass(cls, MLAlpha):
        raise HTTPException(422,
            f"'{req.ml_alpha_name}' is {cls.__name__} (CrossSectionalFactor), "
            f"not an MLAlpha subclass. Use /evaluate-factors for non-ML factors.")

    # 2. Instantiate (UnsupportedEstimatorError / ImportError → 422)
    try:
        alpha = cls()
    except Exception as e:
        raise HTTPException(422,
            f"Failed to instantiate {req.ml_alpha_name}: {type(e).__name__}: {e}")

    # 3. Fetch data (existing helper, warmup-aware lookback)
    end = req.end_date or date.today()
    start = req.start_date or (end - timedelta(days=730))
    try:
        universe_data, calendar = _fetch_data(
            req.symbols, req.market, start, end,
            lookback_days=alpha.warmup_period + 50,
        )
    except Exception as e:
        raise HTTPException(502, f"Data fetch failed: {type(e).__name__}: {e}")

    # 4. Run diagnostics
    config = DiagnosticsConfig(
        forward_horizon=req.forward_horizon,
        severe_overfit_threshold=req.severe_overfit_threshold,
        mild_overfit_threshold=req.mild_overfit_threshold,
        high_turnover_threshold=req.high_turnover_threshold,
        top_n_for_turnover=req.top_n_for_turnover,
    )
    diag = MLDiagnostics(alpha, config=config)
    result = diag.run(universe_data, calendar, start, end, req.eval_freq)

    return result.to_dict()
```

**Error handling matrix:**

| Scenario | Status | Response |
|---|---|---|
| sklearn not installed | 422 | `"scikit-learn>=1.5 required: ..."` |
| Class not found | 404 | `"MLAlpha 'X' not found: KeyError(...)"` |
| Class is CrossSectionalFactor but not MLAlpha | 422 | `"'X' is MomentumRank, not an MLAlpha subclass"` |
| Instantiation fails (whitelist/n_jobs) | 422 | `"Failed to instantiate X: UnsupportedEstimatorError: ..."` |
| Data fetch fails | 502 | `"Data fetch failed: ProviderError: ..."` |
| Symbols list empty | 422 | Pydantic validation error |
| Diagnostics runs, insufficient data | 200 | `{"verdict": "insufficient_data", ...}` |
| Diagnostics succeeds | 200 | Full `DiagnosticsResult.to_dict()` |

## Anti-patterns to Avoid

1. **Do NOT add a parallel `/ml-alpha/evaluate` endpoint.** `/evaluate-factors` already handles MLAlpha.
2. **Do NOT import MLAlpha at module level in portfolio.py.** Use lazy imports inside the endpoint. sklearn is optional.
3. **Do NOT use `pytest.skip` for the happy path.** Mock `_fetch_data` so the test runs deterministically.
4. **Do NOT write test files to the real `ml_alphas/` dir.** Use `tmp_path` + monkeypatch.
