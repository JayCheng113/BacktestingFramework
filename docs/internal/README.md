# 内部文档说明

`docs/internal/` 保存历史设计、实施计划、审计记录和 Codex review 输入。它不是当前用户手册，也不是唯一的实现真相源。

## 目录含义

- `plans/`：按日期保存的历史实施计划。计划中的旧路径、旧测试基线、待办项和截图占位用于追溯当时方案，不代表当前代码状态。
- `specs/`：按日期保存的设计草案或实施规格。带有“历史设计”状态的文件保留原始设计语境。
- `audit/`：历史审计请求与审计报告。
- `core-changes/`：核心架构变更记录和阶段性 roadmap。

## 当前入口

当前项目状态以根目录 `README.md`、`CLAUDE.md`、`docs/guide/`、`docs/architecture/` 和各模块 `CLAUDE.md` 为准。需要判断代码现状时，以源码和测试结果优先，内部历史文档只作为背景材料。
