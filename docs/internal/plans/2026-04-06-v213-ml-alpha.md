**Architecture:** MLAlpha is a stateful `CrossSectionalFactor` subclass that holds an internal trained sklearn-compatible model and retrains lazily when the current eval date moves past its validity window. Model persistence is in-memory only (V1); cross-fold state isolation is provided by `portfolio_walk_forward`'s `strategy_factory()` pattern — each fold calls the factory and gets a brand-new `MLAlpha` instance with no fitted state (see Constraint 3 for why this is NOT a `copy.deepcopy` path in the portfolio flow). Purge and embargo are enforced inside `MLAlpha._build_training_panel()` before `model_factory()` is called. StrategyEnsemble wraps a list of inner `PortfolioStrategy` instances, maintains its own per-sub-strategy state (previous-rebalance target weights + hypothetical-return ledger) to support IC / risk-parity modes without engine changes, and combines sub-strategy outputs before returning to the engine. MLDiagnostics operates on a completed MLAlpha instance + universe_data, walking through retrain checkpoints and reporting feature-importance stability + IS/OOS IC decay + turnover.

**Tech Stack:** `scikit-learn>=1.3` (new dep, added under `[project.optional-dependencies].ml`), existing pandas / numpy / scipy / FastAPI / React / ECharts. LightGBM / XGBoost are **deferred to V2.13.1** (sklearn-only in V1 keeps the estimator whitelist small and the open-question surface area manageable).

**Readiness audit:** `docs/internal/audit/v2.13-readiness-audit.md` — infrastructure verified ready with 8 shape-level contract tests (`tests/test_portfolio/test_v213_readiness.py`).

## First-Principles Rationale

Every design decision in this plan is derived from one of four fundamental constraints. When implementing, if you find yourself deviating, re-read this section and check whether the deviation is justified.

### Constraint 1 — Users cannot be trusted to enforce walk-forward

Given a Python function `strategy.generate_signals(data: DataFrame)`, a user who trains an ML model inside `generate_signals` will see the entire DataFrame and fit to future returns. The backtest Sharpe will be meaningless. This is not hypothetical — it is the single most common way ML backtests lie.

**Therefore**: MLAlpha cannot be a function the user writes. It must be a framework component that receives:
- A pure `model_factory` (returns a fresh, unfit estimator)
- A pure `feature_fn` (maps a DataFrame slice to features)
- A pure `target_fn` (maps a DataFrame slice to labels)

The framework controls train/predict split. The user never touches the split. This is the core anti-cheat mechanism.

### Constraint 2 — Anti-lookahead means STRICTLY less-than

V2.12.2 round 1 verified `slice_universe_data` uses `df.index.date < target_date` (strict `<`, not `<=`). MLAlpha inherits this guarantee via `CrossSectionalFactor.compute(sliced, dt)`. BUT this is only the framework slice. MLAlpha's training data construction adds a second requirement: if the prediction target is `returns[t : t + k]`, then any training sample whose feature date is `s` must satisfy `s + k <= t - embargo`, otherwise the training label contains information about the prediction window.

**Therefore**: `purge_days` must be at least `target_horizon` of the label. `embargo_days` adds further safety margin for autocorrelation. V2.13 enforces this at `_build_training_panel()` by slicing the training panel with `training_date <= prediction_date - purge_days - embargo_days`.

### Constraint 3 — sandbox `pickle` / `multiprocessing` / `threading` bans force in-process single-thread design

`ez/agent/sandbox.py:_FORBIDDEN_MODULES` blocks `pickle`, `multiprocessing`, `threading`, `subprocess`. This constrains what user-written ML code can do at runtime:
- User code CANNOT persist models to disk via `joblib.dump` (joblib uses pickle under the hood) or via `pickle.dump`
- User code CANNOT call `sklearn.RandomForestRegressor(n_jobs=-1).fit(...)` (triggers multiprocessing import at fit time)
- User code CANNOT spawn GPU training subprocesses
- User code CANNOT use native `threading.Thread`

Note that "not in `_FORBIDDEN_MODULES`" is NOT the same as "proven safe". sklearn, scipy, numpy pass the current AST import check and BLAS parallelism via native C threads bypasses Python's `threading` module, but V2.13 does NOT claim the sandbox is safe for arbitrary sklearn usage. V1 ships with a hard whitelist of explicitly-verified estimators (see below), not a blanket "any sklearn class works".

**Therefore**:
1. **V1 estimator whitelist**. MLAlpha runtime checks `type(self._model_factory())` against an allow-list at construction time and at every retrain. Initial list: `sklearn.linear_model.{Ridge, Lasso, LinearRegression, ElasticNet}`, `sklearn.tree.DecisionTreeRegressor`, `sklearn.ensemble.RandomForestRegressor` (with `n_jobs=1`), `sklearn.ensemble.GradientBoostingRegressor`. Anything else raises `UnsupportedEstimatorError`. Adding an estimator to the whitelist requires (a) a task adding it, (b) a pickling/deepcopy regression test, (c) a sandbox smoke test. This is the primary safety mechanism.
2. **n_jobs=1 runtime enforcement**. MLAlpha inspects the returned estimator via `getattr(estimator, 'n_jobs', None)` after `model_factory()` is called. If `n_jobs` is set and != 1 and != None, construction raises. This blocks dynamic / wrapped / `**kwargs`-threaded n_jobs values that a pure AST check cannot see. The Phase 4 AST `n_jobs` literal check is kept as a nice-to-have early warning for simple cases, but it is NOT the enforcement — the runtime check is.
3. **V1 model persistence = in-memory only**. Each MLAlpha instance holds its current fitted model in `self._current_model`. Cross-fold isolation in the portfolio walk-forward flow comes from `portfolio_walk_forward` calling `strategy_factory()` ONCE PER FOLD per stage (IS and OOS), and the factory returning a brand-new `MLAlpha` instance with `_current_model = None`. There is **NO `copy.deepcopy()` on the portfolio path** — the factory is the isolation mechanism. (Single-stock `ez/backtest/walk_forward.py` does use `copy.deepcopy(strategy)` per fold, which is a separate code path; V2.13's integration tests cover both pathways explicitly so the distinction cannot drift.) No disk I/O from user code. No pickle from user code.
4. **GPU training is out of V2.13 scope**. Document as "V2.14+ with proper sandbox escape hatch".

### Constraint 4 — Multi-strategy diversification is additive, and the ensemble must compute "if I had run each sub-strategy alone" internally

A single ML alpha overfits because it optimizes one objective on one dataset. Combining multiple weakly-correlated alphas reduces idiosyncratic variance by `√N`. Ensemble is not a separate product — it is the natural expression of "I do not trust any single model".

**Interface reality**: The existing `PortfolioStrategy.generate_weights(universe_data, date, prev_weights, prev_returns)` interface hands the strategy only the **portfolio-level** previous weights and returns. When StrategyEnsemble wraps sub-strategies, the engine still sees one portfolio. The ensemble does NOT receive "what would sub-strategy A's equity curve have been if it had run standalone" — that series does not exist, because sub-strategy A's weights were combined into a portfolio where it may have been overridden or diluted by sub-strategy B.

This constrains what V1 can do automatically:

**Therefore** (StrategyEnsemble V1 design):
1. **V1 supports three modes with different degrees of automation**:
   - **`"equal"`** — arithmetic mean of sub-strategy weight vectors. No historical state needed. **Fully automatic.**
   - **`"manual"`** — user provides a static `ensemble_weights: list[float]` at construction. Still no historical state needed. **Fully automatic once the user has picked weights out-of-band.** (This is the V1 path for "IC-weighted" in practice — the user computes IC using `CrossSectionalEvaluator` ahead of time and plugs the weights in.)
   - **`"ic_weighted"` / `"risk_parity"` (hybrid)** — StrategyEnsemble maintains its OWN per-sub-strategy hypothetical-return ledger in `self.state`. At every rebalance, it records each sub-strategy's target weight vector. On the NEXT rebalance, it uses the fresh `universe_data` slice to reconstruct each sub-strategy's hypothetical return over the period (using close-to-close return of each held symbol weighted by the stored target weights, without costs or slippage). From this synthetic return series it computes per-sub-strategy IC (vs forward return of a benchmark or vs realized portfolio alpha) or inverse-volatility weights. The first `warmup_rebalances` calls fall back to `"equal"` because the ledger is empty.
2. **Known approximation**: the synthetic return series in mode 3 ignores transaction costs, slippage, optimizer overrides, risk-manager reductions, and lot-size rounding. It is strictly a **hypothetical pure-alpha** return used for weighting decisions, NOT a realized PnL. The ensemble documentation must state this loudly. The alternative (running each sub-strategy through a full backtest at every rebalance to get a realistic return series) is an order of magnitude more expensive and is deferred to V2.13.1.
3. **`StrategyEnsemble` is a `PortfolioStrategy` subclass** (not a separate engine mode). The existing engine sees one strategy (the ensemble); internally the ensemble orchestrates sub-strategies and threads their target weights through its own bookkeeping. No engine modification. This preserves the "single strategy param" invariant and lets ensemble composition be arbitrarily nested.

```python
"""V2.13 F8: MLAlpha — walk-forward ML factor framework.

MLAlpha is a stateful CrossSectionalFactor that holds a trained sklearn-
compatible model internally and retrains lazily when the current eval
date moves beyond the model's validity window.

Anti-lookahead is enforced at two layers:
1. The framework calls `compute(sliced_data, dt)` where `sliced_data`
   already excludes dates >= dt (via slice_universe_data, strict `<`).
2. `_build_training_panel()` further excludes training samples whose
   forward-looking labels would leak into the prediction window via
   purge_days + embargo_days margin.

Model persistence is in-memory only (V1). Cross-fold sharing is NOT
supported. In the portfolio walk-forward flow (ez/portfolio/walk_forward.py),
`portfolio_walk_forward` calls `strategy_factory()` once per fold per
stage (IS, OOS) and the factory returns a brand-new MLAlpha instance with
`_current_model = None` — this is the isolation mechanism on the portfolio
path, NOT copy.deepcopy. (Single-stock backtest walk-forward does deepcopy,
which is a separate code path; MLAlpha supports both because the class
is also deepcopy-safe as a generic Python value — see test_ml_alpha_sklearn.py.)

Safety: V1 supports only a hard whitelist of sklearn estimators and
enforces `n_jobs=1` via runtime inspection of the estimator instance
returned by model_factory(). See _SUPPORTED_ESTIMATOR_CLASSES below.

See docs/internal/plans/2026-04-06-v213-ml-alpha.md for design rationale.
"""
from __future__ import annotations

from datetime import date, datetime, timedelta
from typing import Callable, Any

import numpy as np
import pandas as pd

from ez.portfolio.cross_factor import CrossSectionalFactor


FeatureFn = Callable[[pd.DataFrame], pd.DataFrame]
TargetFn = Callable[[pd.DataFrame], pd.Series]
ModelFactory = Callable[[], Any]


class UnsupportedEstimatorError(TypeError):
    """Raised when MLAlpha is constructed with an estimator class that is
    not on the V1 whitelist, or with an instance whose `n_jobs` would
    trigger multiprocessing (blocked by the sandbox)."""


def _build_supported_estimator_set() -> frozenset[type]:
    """Construct the V1 whitelist of sklearn estimator classes.

    Built lazily (not at module import) so that `import ez.portfolio.ml_alpha`
    works even when sklearn is not installed — only the first MLAlpha
    construction triggers the sklearn import.

    Adding a class to this set is NOT free: it requires
    (1) a deepcopy/determinism regression test in test_ml_alpha_sklearn.py
    (2) a sandbox smoke test confirming import + fit under the sandbox
    (3) a task in the plan explicitly calling out the addition.
    """
    try:
        from sklearn.linear_model import (
            Ridge, Lasso, LinearRegression, ElasticNet,
        )
        from sklearn.tree import DecisionTreeRegressor
        from sklearn.ensemble import (
            RandomForestRegressor, GradientBoostingRegressor,
        )
    except ImportError as e:
        raise ImportError(
            "scikit-learn>=1.3 is required for MLAlpha. "
            "Install with: pip install -e '.[ml]'"
        ) from e
    return frozenset({
        Ridge, Lasso, LinearRegression, ElasticNet,
        DecisionTreeRegressor,
        RandomForestRegressor, GradientBoostingRegressor,
    })


_SUPPORTED_ESTIMATOR_CACHE: frozenset[type] | None = None


def _assert_supported_estimator(instance: Any) -> None:
    """Enforce the V1 estimator whitelist + n_jobs=1 rule at runtime.

    Uses `type(instance)` identity comparison rather than `isinstance` to
    avoid accidentally accepting user subclasses that might override fit()
    with unsafe behavior. If an advanced user legitimately needs a subclass,
    they can add the subclass to the whitelist explicitly.
    """
    global _SUPPORTED_ESTIMATOR_CACHE
    if _SUPPORTED_ESTIMATOR_CACHE is None:
        _SUPPORTED_ESTIMATOR_CACHE = _build_supported_estimator_set()

    cls = type(instance)
    if cls not in _SUPPORTED_ESTIMATOR_CACHE:
        raise UnsupportedEstimatorError(
            f"Estimator class {cls.__module__}.{cls.__name__} is not on "
            f"the V1 MLAlpha whitelist. Supported classes: "
            f"{sorted(c.__name__ for c in _SUPPORTED_ESTIMATOR_CACHE)}. "
            f"If you need another estimator, add it to the whitelist with "
            f"(a) a deepcopy regression test, (b) a sandbox smoke test, "
            f"and (c) an explicit plan-file entry."
        )

    n_jobs = getattr(instance, "n_jobs", None)
    if n_jobs is not None and n_jobs != 1:
        raise UnsupportedEstimatorError(
            f"Estimator {cls.__name__} has n_jobs={n_jobs}, but the sandbox "
            f"blocks `multiprocessing`. Construct with n_jobs=1 explicitly."
        )


class MLAlpha(CrossSectionalFactor):
    """Walk-forward ML factor. See module docstring for design.

    Args:
        name: Unique factor name (e.g., "ridge_momentum_v1").
        model_factory: Callable returning a fresh unfit estimator.
            MUST return a NEW instance each call (framework will call
            it multiple times across retrain boundaries).
        feature_fn: Per-symbol feature extractor. Called with a
            single-symbol DataFrame (sliced to training window),
            returns a DataFrame of features indexed by date.
        target_fn: Per-symbol target extractor. Called with the same
            single-symbol DataFrame, returns a Series of forward-
            looking labels aligned to the feature dates.
        train_window: Number of trailing trading days to use for each
            retrain's training panel.
        retrain_freq: Retrain when current prediction date exceeds last
            retrain date by this many calendar days.
        purge_days: Exclude the last N training dates before the
            prediction date to prevent feature-label temporal overlap.
            MUST be at least the target's forward horizon.
        embargo_days: Additional buffer on top of purge_days. Defaults
            to 0 (purge is already the minimum safe gap).
    """

    def __init__(
        self,
        name: str,
        model_factory: ModelFactory,
        feature_fn: FeatureFn,
        target_fn: TargetFn,
        train_window: int,
        retrain_freq: int,
        purge_days: int,
        embargo_days: int = 0,
    ):
        if train_window <= 0:
            raise ValueError(f"train_window must be > 0, got {train_window}")
        if retrain_freq <= 0:
            raise ValueError(f"retrain_freq must be > 0, got {retrain_freq}")
        if purge_days < 0:
            raise ValueError(f"purge_days must be >= 0, got {purge_days}")
        if embargo_days < 0:
            raise ValueError(f"embargo_days must be >= 0, got {embargo_days}")

        self._name = name
        self._model_factory = model_factory
        self._feature_fn = feature_fn
        self._target_fn = target_fn
        self._train_window = train_window
        self._retrain_freq = retrain_freq
        self._purge_days = purge_days
        self._embargo_days = embargo_days

        # V1 safety: validate the estimator BEFORE any fit happens. Raises
        # UnsupportedEstimatorError if the class is not on the whitelist or
        # if n_jobs is set to something other than 1/None. This runs once at
        # construction so the user gets an immediate failure, not a
        # mysterious multiprocessing crash deep inside fit().
        _assert_supported_estimator(model_factory())

        # Runtime state. A fresh MLAlpha instance (from strategy_factory())
        # starts with _current_model=None, so portfolio walk-forward gets
        # per-fold isolation "for free". copy.deepcopy also works (supported
        # as a generic Python value) for single-stock WalkForwardValidator
        # compatibility.
        self._current_model: Any = None
        self._last_retrain_date: date | None = None
        self._retrain_count: int = 0

    @property
    def name(self) -> str:
        return self._name

    @property
    def warmup_period(self) -> int:
        # Need at least train_window + purge + embargo days of history
        # before the first prediction date can be made.
        return self._train_window + self._purge_days + self._embargo_days

    def compute(self, universe_data: dict[str, pd.DataFrame], dt: datetime) -> pd.Series:
        """Return per-symbol predictions at date dt."""
        # To be implemented in task 1.4
        return pd.Series(dtype=float)

    def compute_raw(self, universe_data: dict[str, pd.DataFrame], dt: datetime) -> pd.Series:
        """Raw predictions (no ranking). Used by AlphaCombiner."""
        return self.compute(universe_data, dt)
```

```python
    def _build_training_panel(
        self,
        universe_data: dict[str, pd.DataFrame],
        prediction_date: date,
    ) -> tuple[pd.DataFrame | None, pd.Series | None]:
        """Stack per-symbol (feature, target) pairs into a cross-sectional
        training panel.

        ⚠️ CRITICAL (V2.13 review round fix C1): purge_days / embargo_days
        are measured in TRADING DAYS (data rows), not calendar days. Reason:
        typical target_fn shapes like `pct_change(5).shift(-5)` use a
        trading-day-unit forward horizon (pandas shift operates on rows).
        If we trimmed by calendar days via `timedelta(days=N)`, a weekend
        would span 2 calendar days but 0 trading days, so the last 2 rows
        of training labels would point INTO the prediction window → label
        leakage. Positional trim (`iloc[:-purge_bars]`) matches shift's
        row-based unit, so label at the retained tail cannot reach into
        [prediction_date, prediction_date + k).

        Exclusion rules (per symbol):
        1. Strict anti-lookahead on feature date: feat.date < prediction_date
        2. Positional purge on tail: drop last (purge_days + embargo_days)
           rows (trading days)
        3. NaN target filter: shift(-k) produces a NaN tail; drop those
        4. Cap at last `train_window` rows per symbol
        """
        purge_bars = self._purge_days + self._embargo_days

        rows: list[pd.DataFrame] = []
        labels: list[pd.Series] = []
        for sym, df in universe_data.items():
            if not isinstance(df.index, pd.DatetimeIndex):
                continue
            # Extract features and target for this symbol
            try:
                sym_features = self._feature_fn(df)
                sym_target = self._target_fn(df)
            except Exception:
                continue
            if sym_features is None or sym_target is None:
                continue
            # Type check + log warning once (I3 fix)
            if not isinstance(sym_features, pd.DataFrame):
                # (warning log logic) - skip
                continue
            if not isinstance(sym_target, pd.Series):
                continue
            if sym_features.empty or sym_target.empty:
                continue

            # Align features and target on date index
            aligned_idx = sym_features.index.intersection(sym_target.index)
            if len(aligned_idx) == 0:
                continue
            feat = sym_features.loc[aligned_idx]
            tgt = sym_target.loc[aligned_idx]

            # Step 1: strict anti-lookahead — feature date < prediction_date
            strict_mask = feat.index.date < prediction_date
            feat = feat.loc[strict_mask]
            tgt = tgt.loc[strict_mask]
            if len(feat) == 0:
                continue

            # Step 2: POSITIONAL purge (trading days, NOT calendar days).
            # This is the fix for review round C1.
            if purge_bars > 0:
                if len(feat) <= purge_bars:
                    continue  # not enough rows to purge safely
                feat = feat.iloc[:-purge_bars]
                tgt = tgt.iloc[:-purge_bars]

            # Step 3: drop NaN targets (shift tail already excluded by
            # step 2, but pct_change warmup can still leave NaN)
            valid = ~tgt.isna()
            feat = feat.loc[valid]
            tgt = tgt.loc[valid]

            if len(feat) == 0:
                continue

            # Step 4: cap at train_window rows
            if len(feat) > self._train_window:
                feat = feat.iloc[-self._train_window:]
                tgt = tgt.iloc[-self._train_window:]

            # Add symbol as a MultiIndex level for panel assembly
            feat = feat.copy()
            feat.index = pd.MultiIndex.from_arrays(
                [feat.index, [sym] * len(feat)],
                names=["date", "symbol"],
            )
            tgt = tgt.copy()
            tgt.index = feat.index

            rows.append(feat)
            labels.append(tgt)

        if not rows:
            return None, None

        X = pd.concat(rows, axis=0)
        y = pd.concat(labels, axis=0)
        # Drop any remaining NaN rows in X. inf filtering is done in
        # _retrain (I1 fix).
        valid_rows = ~X.isna().any(axis=1)
        X = X.loc[valid_rows]
        y = y.loc[valid_rows]

        if len(X) == 0:
            return None, None
        return X, y
```

```python
    def _predict(
        self,
        universe_data: dict[str, pd.DataFrame],
        current: date,
    ) -> pd.Series:
        """Predict scores for each symbol at the current date.

        For each symbol, extract the most recent feature row (strictly
        before `current` — anti-lookahead is already enforced by the
        engine via slice_universe_data, but we double-check here).
        """
        if self._current_model is None:
            return pd.Series(dtype=float)

        predictions: dict[str, float] = {}
        for sym, df in universe_data.items():
            if not isinstance(df.index, pd.DatetimeIndex):
                continue
            try:
                sym_features = self._feature_fn(df)
            except Exception:
                continue
            if sym_features is None or sym_features.empty:
                continue

            # Take the latest feature row that is strictly before `current`
            mask = sym_features.index.date < current
            if not mask.any():
                continue
            latest_features = sym_features.loc[mask].iloc[-1:]
            if latest_features.isna().any().any():
                continue

            try:
                pred = float(self._current_model.predict(latest_features.values)[0])
            except Exception:
                continue
            predictions[sym] = pred

        return pd.Series(predictions, dtype=float) if predictions else pd.Series(dtype=float)
```

```python
"""V2.13 D5: StrategyEnsemble — multi-strategy long-only combiner.

Wraps a list of PortfolioStrategy instances and combines their outputs
using equal / manual / ic_weighted / risk_parity weighting. Output is a
single dict[symbol → weight] compatible with the existing engine's
optimizer and allocator layers — no engine changes required.

Design constraint (see plan Constraint 4):
The PortfolioStrategy interface hands `generate_weights()` only the
portfolio-level prev_weights / prev_returns. To compute per-sub-strategy
IC or risk-parity weights, the ensemble must reconstruct each sub-
strategy's hypothetical return INTERNALLY from its own state ledger.

State ledger (maintained in self.state):
    {
        "sub_target_weights": [
            {"rebalance_idx": int, "weights": dict[sym, float], "date": date},
            ...  # one record per sub-strategy per rebalance
        ]  × len(strategies),
        "sub_hypothetical_returns": list[list[float]],
            # For each sub-strategy, a list of close-to-close returns
            # computed from the stored target weights between consecutive
            # rebalances. Ignores costs/slippage/risk-manager/optimizer
            # overrides — this is "pure alpha" assumed PnL, NOT realized.
        "last_rebalance_date": date | None,
    }

The "ic_weighted" and "risk_parity" modes require at least
`warmup_rebalances` data points in `sub_hypothetical_returns[i]` before
switching from equal-weight fallback to computed weights.
"""
from __future__ import annotations

from datetime import date as _date, datetime
from typing import Literal

import numpy as np
import pandas as pd

from ez.portfolio.portfolio_strategy import PortfolioStrategy


EnsembleMode = Literal["equal", "manual", "ic_weighted", "risk_parity"]


class StrategyEnsemble(PortfolioStrategy):
    """Combine multiple sub-strategies into a single weight vector.

    Args:
        strategies: List of sub-strategies. Each must implement
            `generate_weights()`. Called in order at each rebalance.
        mode: Combination mode.
            - "equal": arithmetic mean of sub-strategy weight vectors.
              Stateless — fully automatic.
            - "manual": uses `ensemble_weights` (required). Stateless —
              fully automatic once the user picks weights out-of-band.
              Intended for users who compute IC via CrossSectionalEvaluator
              ahead of time and pass a pre-baked weight vector.
            - "ic_weighted": maintains per-sub-strategy hypothetical-return
              ledger internally, computes rank-IC vs the next-period
              symbol-wise forward return, and weights sub-strategies by
              normalized |IC|. Falls back to equal weights for the first
              `warmup_rebalances` rebalances.
            - "risk_parity": same ledger, but weights = 1/vol normalized
              (inverse-volatility). Falls back to equal weights during
              warmup.
        ensemble_weights: Required for mode="manual". Length must match
            `strategies`. Must be non-negative and sum to > 0 (will be
            normalized).
        warmup_rebalances: Minimum number of ledger entries before
            ic_weighted/risk_parity switch from equal-weight fallback.
            Default 8 (approximately 2 months of weekly rebalance).
    """

    # NOTE: V1 scope — "ic_weighted" and "risk_parity" use an approximation
    # of per-sub-strategy return (pure-alpha, ignores costs/slippage/risk
    # manager / optimizer overrides). Full realized-return ensemble is a
    # V2.13.1 follow-up that requires running a sub-backtest per rebalance.

    def __init__(
        self,
        strategies: list[PortfolioStrategy],
        mode: EnsembleMode = "equal",
        ensemble_weights: list[float] | None = None,
        warmup_rebalances: int = 8,
    ):
        super().__init__()
        if not strategies:
            raise ValueError("StrategyEnsemble requires at least one strategy")
        if mode not in ("equal", "manual", "ic_weighted", "risk_parity"):
            raise ValueError(f"Invalid mode: {mode}")
        if mode == "manual":
            if ensemble_weights is None:
                raise ValueError('mode="manual" requires ensemble_weights')
            if len(ensemble_weights) != len(strategies):
                raise ValueError(
                    f"ensemble_weights length {len(ensemble_weights)} does "
                    f"not match strategies length {len(strategies)}"
                )
            if any(w < 0 for w in ensemble_weights):
                raise ValueError("ensemble_weights must be non-negative")
            if sum(ensemble_weights) <= 0:
                raise ValueError("ensemble_weights must sum to > 0")
        if warmup_rebalances < 1:
            raise ValueError("warmup_rebalances must be >= 1")

        self._strategies = strategies

        sub_outputs: list[dict[str, float]] = []
        for s in self._strategies:
```

```python
def _update_hypothetical_returns(
    self,
    universe_data: dict[str, pd.DataFrame],
    current: _date,
) -> None:
    """For each sub-strategy, append one hypothetical return for the
    period since its most recently recorded rebalance.

    Formula: r_i = sum_sym w_sym * (close_sym(current) / close_sym(prev_rebalance) - 1)
    where w_sym is the sub-strategy's target weight at the previous
    rebalance and close_sym is the anti-lookahead-safe close at each date.

    Approximation: no costs, no slippage, no risk-manager reduction,
    no optimizer override. Documented in module docstring.
    """
    for i, sub_records in enumerate(self.state["sub_target_weights"]):
        if not sub_records:
            continue
        prev = sub_records[-1]
        prev_date = prev["date"]
        prev_weights = prev["weights"]
        if prev_date >= current:
            continue  # current is the first rebalance or re-entry
        if not prev_weights:
            # Sub-strategy held nothing — hypothetical return is 0
            self.state["sub_hypothetical_returns"][i].append(0.0)
            continue

        period_return = 0.0
        total_weight = 0.0
        for sym, w in prev_weights.items():
            if w <= 0 or sym not in universe_data:
                continue
            df = universe_data[sym]
            if not isinstance(df.index, pd.DatetimeIndex):
                continue
            # Find the close prices at prev_date and (current - 1 day)
            # using strict anti-lookahead (< current). This aligns with
            # slice_universe_data's < target_date convention.
            mask_current = df.index.date < current
            mask_prev = df.index.date <= prev_date
            if not mask_current.any() or not mask_prev.any():
                continue
            # close at previous rebalance (inclusive of prev_date)
            prev_close_rows = df.loc[mask_prev & (df.index.date >= prev_date)]
            if prev_close_rows.empty:
                continue
            prev_close = float(prev_close_rows["close"].iloc[0])
            # close immediately before current rebalance
            current_close = float(df.loc[mask_current]["close"].iloc[-1])
            if prev_close <= 0:
                continue
            sym_ret = current_close / prev_close - 1.0
            period_return += w * sym_ret
            total_weight += w

        # If the sub-strategy's weights didn't sum to 1, the remainder was
        # implicit cash — contributes 0 return. Leave period_return as-is.
        self.state["sub_hypothetical_returns"][i].append(period_return)
```

```python
def _ic_weighted_or_fallback(self) -> list[float]:
    """Rank-IC weighted. For each sub-strategy, compute the Spearman rank
    correlation between its hypothetical returns and the cross-sectional
    rank of those returns across sub-strategies at each rebalance. A
    sub-strategy with high IC tends to produce returns that rank well
    at times when the ensemble's signal is strong.

    V1 simplification: weight_i ∝ max(0, mean of sub_i's hypothetical
    returns) — the sub-strategy with the highest realized pure-alpha
    return gets the largest weight. This is the conservative form of
    IC weighting and matches the existing AlphaCombiner "ic" mode.
    """
    n = len(self._strategies)
    returns_lists = self.state["sub_hypothetical_returns"]
    min_len = min(len(r) for r in returns_lists) if returns_lists else 0
    if min_len < self._warmup:
        return [1.0] * n  # equal-weight fallback during warmup

    raw = []
    for r_list in returns_lists:
        recent = r_list[-max(self._warmup, 12):]
        mean_r = float(np.mean(recent)) if recent else 0.0
        raw.append(max(0.0, mean_r))

    if sum(raw) <= 0:
        return [1.0] * n  # all non-positive → equal fallback
    return raw
```

```python
def _risk_parity_or_fallback(self) -> list[float]:
    """Inverse-volatility weighting over the hypothetical-return ledger.

    weight_i ∝ 1 / max(std(sub_i's hypothetical returns), epsilon)

    V1 does NOT use a full covariance matrix (which would require the
    ensemble to align rebalance dates across subs — the current design
    guarantees alignment because all subs are called at every rebalance).
    Pure inverse-vol is a well-known risk-parity approximation that is
    exact only when sub-strategies are uncorrelated; the ensemble
    documents this and defers full covariance risk-parity to V2.13.1.
    """
    n = len(self._strategies)
    returns_lists = self.state["sub_hypothetical_returns"]
    min_len = min(len(r) for r in returns_lists) if returns_lists else 0
    if min_len < self._warmup:
        return [1.0] * n

    eps = 1e-6
    raw = []
    for r_list in returns_lists:
        recent = r_list[-max(self._warmup, 12):]
        sd = float(np.std(recent, ddof=1)) if len(recent) > 1 else 0.0
        raw.append(1.0 / max(sd, eps))
    return raw
```

```python
def _check_n_jobs_literal_warning(tree: ast.AST) -> list[str]:
    """Scan for n_jobs keyword arguments with literal values != 1.

    This is a best-effort early warning for sandbox users. It catches
    the simple case `RandomForestRegressor(n_jobs=-1)` so the user sees
    a message in the editor before runtime. It does NOT catch:
    - Variable-bound n_jobs: `n = -1; Rf(n_jobs=n)`
    - Kwarg spread: `Rf(**config)`
    - Wrapped constructors: `make_rf(cores=-1)`
    - Runtime-dispatch: `cls(**{'n_jobs': -1})`

    The real safety is `MLAlpha._assert_supported_estimator` in
    `ez/portfolio/ml_alpha.py`, which inspects the instance returned
    by `model_factory()` at construction time via
    `getattr(instance, 'n_jobs', None)`. Do not rely on this AST check
    for correctness.
    """
    warnings = []
    for node in ast.walk(tree):
        if isinstance(node, ast.Call):
            for kw in node.keywords:
                if kw.arg == "n_jobs":
                    if isinstance(kw.value, ast.Constant):
                        if kw.value.value not in (1, None):
                            warnings.append(
                                f"Line {node.lineno}: n_jobs={kw.value.value} "
                                f"will raise UnsupportedEstimatorError at "
                                f"MLAlpha construction (sandbox blocks "
                                f"`multiprocessing`). Set n_jobs=1 explicitly."
                            )
                    elif isinstance(kw.value, ast.UnaryOp):
                        if isinstance(kw.value.operand, ast.Constant):
                            val = kw.value.operand.value
                            if isinstance(kw.value.op, ast.USub):
                                val = -val
                            if val != 1:
                                warnings.append(
                                    f"Line {node.lineno}: n_jobs={val} "
                                    f"will raise at MLAlpha construction."
                                )
    return warnings
```
