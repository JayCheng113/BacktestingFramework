**目标**：在 Web 端内嵌类似 Claude Code 的 AI 助手，辅助人类研究员更快地写策略、调试、理解结果。**人类驾驶，Agent 副驾**。

### LLM Provider 抽象层 (`ez/llm/`)

支持多家 LLM，**中国国内可用为首要考虑**。

| Provider | 国内可用 | tool_use/function_calling | 优先级 |
|----------|---------|--------------------------|--------|
| **DeepSeek** | ✅ 直连 | ✅ OpenAI 兼容格式 | **P0** |
| **Qwen (通义千问)** | ✅ 直连 | ✅ 原生 tool_use | P1 |
| Claude (Anthropic) | ❌ 需代理 | ✅ tool_use | P2 |
| OpenAI | ❌ 需代理 | ✅ function_calling | P2 |
| 本地模型 (Ollama/vLLM) | ✅ 完全离线 | ⚠️ 部分支持 | P1 |

```python
# ez/llm/provider.py
class LLMProvider(ABC):
    def chat(self, messages, tools=None) -> LLMResponse: ...
    def stream_chat(self, messages, tools=None) -> Iterator[LLMEvent]: ...

class DeepSeekProvider(LLMProvider): ...   # 默认
class QwenProvider(LLMProvider): ...
class AnthropicProvider(LLMProvider): ...
class OpenAICompatibleProvider(LLMProvider): ...  # 覆盖 Ollama/vLLM
```

```yaml
# config.yaml
llm:
  provider: deepseek           # deepseek | qwen | anthropic | openai | local
  api_key: ${DEEPSEEK_API_KEY}
  model: deepseek-chat
  base_url: ~                  # 留空用默认，本地模型填 http://localhost:11434
```

## V2.8 — Autonomous Research Agent (自主研究 Agent 流水线)

**目标**：Agent 自主探索策略空间 — 生成假设、写代码、批量实验、评估筛选、迭代优化。**Agent 驾驶，人类审核**。

**扩展模块**：ez/agent/ (hypothesis.py, code_gen.py, loop_controller.py)

> **与 V2.7 Web Assistant 的本质区别**：
> - V2.7 = 同步、交互式、每步人类确认
> - V2.8 = 异步、自主式、人类只审核最终批次结果

### 研究 Agent 流水线

```
人类设定目标
  "探索 A 股日线动量类策略，Sharpe > 1，回撤 < 20%"
     │
     ▼
┌─────────────────────────────────────────┐
│  Hypothesis Generator (LLM)             │
│  输入: 目标 + 已有策略 + 市场数据概况    │
│  输出: N 个策略假设 (自然语言描述)       │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Code Generator (LLM)                    │
│  输入: 假设 + Strategy ABC 接口          │
│  输出: N 个 .py 策略文件                 │
│  → 自动 contract test → 过滤编译失败     │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Batch Runner (已有 V2.4 Runner)         │
│  对每个策略执行: backtest + WFO + sig    │
│  → ExperimentStore 持久化                │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Research Gate (已有 V2.4 Gate)           │
│  过滤: Sharpe/DD/trades/significance     │
│  → 淘汰不达标策略                        │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Analyzer (LLM)                          │
│  输入: 通过/失败的策略 + metrics          │
│  输出: 改进建议 / 新假设方向              │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Loop Controller                         │
│  决策: 继续迭代? 尝试新方向? 停止?       │
│  条件: 达到目标 / 超出预算 / N轮无进展   │
└──────────────┬──────────────────────────┘
               ▼
        研究报告 → 人类审核
```

### 与 V2.5 Scale 的区别

| 维度 | V2.5 Scale | V2.8 Research Agent |
|------|-----------|-------------------|
| 搜索方式 | 参数网格/随机搜索（固定策略结构） | LLM 生成新策略结构 |
| 创造性 | 无（只调参数） | 有（生成全新逻辑） |
| 人类参与 | 设定搜索空间 | 设定研究目标 |
| 输出 | 最优参数组合 | 新策略代码 + 研究报告 |

## V2.9 — Portfolio / Rotation (多股组合回测)

**目标**：从"单股票单策略"升级到"多股票组合/轮动"回测。

**设计文档**：`docs/internal/core-changes/portfolio-design.md`

**新模块**：ez/portfolio/ (universe, cross_factor, portfolio_strategy, allocator, engine, metrics)

> **设计决策（第一性原理审查 + Codex 审计修正）**：
> - **防前瞻（P0 Codex #2）**：引擎在调用 generate_weights 前**切片**数据至 `[date - lookback_days, date - 1]`。策略函数**不可能**看到当日或未来数据。不依赖 shift(1)——数据入口本身就是安全的。
> - **PIT Universe（P0 Codex #3）**：V2.9 必做。动态成分股 + 退市/IPO 处理 + 交易日历，消除幸存者偏差。不后置。
> - **会计不变量（P0 Codex #4）**：PortfolioEngine 用离散股数（不是连续权重）记账。每日强制 `cash + Σ(shares × price) == equity`，违反即 assert 失败。佣金/印花税/最低手续费在成交时扣减。
> - **交易日历（P1 Codex #8）**：引入 TradingCalendar 模块（至少沪深交易所），所有调仓/对齐都走它。不用 week number 分组。
> - **多时间框架**：不在引擎层做。策略拿到的是已切片的历史日线，内部自行 resample。
> - **有状态策略**：必须支持。ABC 提供 `self.state` + `prev_weights`/`prev_returns` 参数。
> - **ETF 数据**：ETF 不是特殊类型。Universe 接受任何可交易证券代码。
> - **复合策略**：层级依赖在单个 PortfolioStrategy 内部实现。
> - **涨跌停执行过滤**：属于引擎层。策略输出理想权重，引擎根据可交易性调整。
> - **Long-only 约束**：A 股不可做空。引擎强制 weight >= 0。
> - **实施顺序（P1 Codex #9）**：先引擎后入口。Phase A: P1-P7（引擎+API），Phase B: P8-P11（编辑器+Agent+前端）。Phase A 不依赖 Phase B。
