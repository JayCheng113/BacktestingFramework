### 3. PortfolioStrategy + Allocator + Optimizer composition — READY

**Requirement**: V2.13 StrategyEnsemble combines multiple
`PortfolioStrategy` instances.

**Evidence**:
- `run_portfolio_backtest()` accepts a single `strategy` parameter.
- `PortfolioStrategy` ABC is stateful — strategies can hold sub-strategies.
- `Allocator` operates on a single weight dict (post-strategy, post-optimizer).
- `PortfolioOptimizer` (V2.12) takes raw alpha weights and returns
  optimized weights — also single-dict interface.

**Design path for V2.13 StrategyEnsemble**:
```python
class StrategyEnsemble(PortfolioStrategy):
    def __init__(self, strategies: list[PortfolioStrategy], weights: list[float]):
        self._strategies = strategies
        self._weights = weights

    def generate_weights(self, data, date, prev_w, prev_r):
        sub_weights = [s.generate_weights(data, date, prev_w, prev_r)
                       for s in self._strategies]
        # Combine via weighted sum
        combined: dict[str, float] = {}
        for sw, w in zip(sub_weights, self._weights):
            for sym, wgt in sw.items():
                combined[sym] = combined.get(sym, 0.0) + w * wgt
        return combined
```

Engine sees a single `strategy` (the ensemble). No engine modification
needed. This matches the existing stateful-strategy pattern.

**Verdict**: READY. Ensemble is a `PortfolioStrategy` subclass; no new
engine parameters or allocator changes required.

### 5. Data layer lookback propagation — READY

**Requirement**: V2.13 ML alphas may use 250-500+ day train windows.

**Evidence** (verified via direct calculation):
- `_max_factor_warmup(factors)` in `ez/api/routes/portfolio.py:175-202`
  computes `max(warmup) + 50` across factor instances.
- `_fetch_data(..., lookback_days=dynamic_lb)` at line 466-491 fetches
  `start - dynamic_lb * 1.6` of history.
- Round 5 propagation fix: `dynamic_lb` passed to
  `evaluate_cross_sectional_factor`, `evaluate_ic_decay`, and
  `compute_factor_correlation` (prior version only lengthened the fetch,
  leaving evaluators with their default 252).

**Concrete verification**:
```
backtest_start = 2024-01-01, factor_warmup = 504, buffer = 50
dynamic_lb = 554
train_start = 2024-01-01 - 365 = 2023-01-01  (AlphaCombiner train window)
fetch_start = train_start - 554*1.6 = 2020-07-29
first_eval_slice = [2023-01-01 - 554, 2022-12-31] = [2021-06-26, 2022-12-31]
fetch_start (2020-07-29) <= first_slice_start (2021-06-26) ✓
Total history fetched: 1251 days ≈ 3.4 years
```

**Note**: The hardcoded 365 days in `_compute_alpha_weights` is the
**IC evaluation time range** (how many weekly eval dates to use for IC
statistics), not a factor lookback. At `eval_freq="weekly"`, 365 days
yields ~52 IC samples — sufficient for statistical weighting. This is
NOT a V2.13 blocker; it's a separate semantic from factor lookback
which is already dynamic.

**Verdict**: READY. 500+ day lookback supported without changes.

   from `CrossSectionalFactor`. Minimal: accepts `model_factory`,

   + IS/OOS IC decay + turnover analysis. Pure compute, no new abstractions.

   wrapper that combines sub-strategy outputs. Equal / risk-parity /
   IC-weighted modes.

6. **API**: `/api/portfolio/ml-alpha/evaluate` endpoint mirroring
   `/evaluate-factors` for MLAlpha instances.
