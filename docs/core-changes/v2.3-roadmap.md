**目标**：在 Web 端内嵌类似 Claude Code 的 AI 助手，辅助人类研究员更快地写策略、调试、理解结果。**人类驾驶，Agent 副驾**。

> - Web Assistant (V2.7) = 人类说"帮我写个 RSI 策略"，Agent 写代码、跑测试、人类确认

### LLM Provider 抽象层 (`ez/llm/`)

支持多家 LLM，**中国国内可用为首要考虑**。

| Provider | 国内可用 | tool_use/function_calling | 优先级 |
|----------|---------|--------------------------|--------|
| **DeepSeek** | ✅ 直连 | ✅ OpenAI 兼容格式 | **P0** |
| **Qwen (通义千问)** | ✅ 直连 | ✅ 原生 tool_use | P1 |
| Claude (Anthropic) | ❌ 需代理 | ✅ tool_use | P2 |
| OpenAI | ❌ 需代理 | ✅ function_calling | P2 |
| 本地模型 (Ollama/vLLM) | ✅ 完全离线 | ⚠️ 部分支持 | P1 |

```python
# ez/llm/provider.py
class LLMProvider(ABC):
    def chat(self, messages, tools=None) -> LLMResponse: ...
    def stream_chat(self, messages, tools=None) -> Iterator[LLMEvent]: ...

class DeepSeekProvider(LLMProvider): ...   # 默认
class QwenProvider(LLMProvider): ...
class AnthropicProvider(LLMProvider): ...
class OpenAICompatibleProvider(LLMProvider): ...  # 覆盖 Ollama/vLLM
```

```yaml
# config.yaml
llm:
  provider: deepseek           # deepseek | qwen | anthropic | openai | local
  api_key: ${DEEPSEEK_API_KEY}
  model: deepseek-chat
  base_url: ~                  # 留空用默认，本地模型填 http://localhost:11434
```

## V2.8 — Autonomous Research Agent (自主研究 Agent 流水线)

**目标**：Agent 自主探索策略空间 — 生成假设、写代码、批量实验、评估筛选、迭代优化。**Agent 驾驶，人类审核**。

**扩展模块**：ez/agent/ (hypothesis.py, code_gen.py, loop_controller.py)

> **与 V2.7 Web Assistant 的本质区别**：
> - V2.7 = 同步、交互式、每步人类确认
> - V2.8 = 异步、自主式、人类只审核最终批次结果

### 研究 Agent 流水线

```
人类设定目标
  "探索 A 股日线动量类策略，Sharpe > 1，回撤 < 20%"
     │
     ▼
┌─────────────────────────────────────────┐
│  Hypothesis Generator (LLM)             │
│  输入: 目标 + 已有策略 + 市场数据概况    │
│  输出: N 个策略假设 (自然语言描述)       │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Code Generator (LLM)                    │
│  输入: 假设 + Strategy ABC 接口          │
│  输出: N 个 .py 策略文件                 │
│  → 自动 contract test → 过滤编译失败     │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Batch Runner (已有 V2.4 Runner)         │
│  对每个策略执行: backtest + WFO + sig    │
│  → ExperimentStore 持久化                │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Research Gate (已有 V2.4 Gate)           │
│  过滤: Sharpe/DD/trades/significance     │
│  → 淘汰不达标策略                        │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Analyzer (LLM)                          │
│  输入: 通过/失败的策略 + metrics          │
│  输出: 改进建议 / 新假设方向              │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Loop Controller                         │
│  决策: 继续迭代? 尝试新方向? 停止?       │
│  条件: 达到目标 / 超出预算 / N轮无进展   │
└──────────────┬──────────────────────────┘
               ▼
        研究报告 → 人类审核
```

### 与 V2.5 Scale 的区别

| 维度 | V2.5 Scale | V2.8 Research Agent |
|------|-----------|-------------------|
| 搜索方式 | 参数网格/随机搜索（固定策略结构） | LLM 生成新策略结构 |
| 创造性 | 无（只调参数） | 有（生成全新逻辑） |
| 人类参与 | 设定搜索空间 | 设定研究目标 |
| 输出 | 最优参数组合 | 新策略代码 + 研究报告 |

## V2.9 — Portfolio / Rotation (多股组合回测)

**目标**：从"单股票单策略"升级到"多股票组合/轮动"回测。

**设计文档**：`docs/core-changes/portfolio-design.md`

**新模块**：ez/portfolio/ (universe, cross_factor, portfolio_strategy, allocator, engine, metrics)

> **设计决策（第一性原理审查 + Codex 审计修正）**：
> - **防前瞻（P0 Codex #2）**：引擎在调用 generate_weights 前**切片**数据至 `[date - lookback_days, date - 1]`。策略函数**不可能**看到当日或未来数据。不依赖 shift(1)——数据入口本身就是安全的。
> - **PIT Universe（P0 Codex #3）**：V2.9 必做。动态成分股 + 退市/IPO 处理 + 交易日历，消除幸存者偏差。不后置。
> - **会计不变量（P0 Codex #4）**：PortfolioEngine 用离散股数（不是连续权重）记账。每日强制 `cash + Σ(shares × price) == equity`，违反即 assert 失败。佣金/印花税/最低手续费在成交时扣减。
> - **交易日历（P1 Codex #8）**：引入 TradingCalendar 模块（至少沪深交易所），所有调仓/对齐都走它。不用 week number 分组。
> - **多时间框架**：不在引擎层做。策略拿到的是已切片的历史日线，内部自行 resample。
> - **有状态策略**：必须支持。ABC 提供 `self.state` + `prev_weights`/`prev_returns` 参数。
> - **ETF 数据**：ETF 不是特殊类型。Universe 接受任何可交易证券代码。
> - **复合策略**：层级依赖在单个 PortfolioStrategy 内部实现。
> - **涨跌停执行过滤**：属于引擎层。策略输出理想权重，引擎根据可交易性调整。
> - **Long-only 约束**：A 股不可做空。引擎强制 weight >= 0。
> - **实施顺序（P1 Codex #9）**：先引擎后入口。Phase A: P1-P7（引擎+API），Phase B: P8-P11（编辑器+Agent+前端）。Phase A 不依赖 Phase B。

**目标**：接入 A 股基本面数据（多因子选股的核心数据源）+ 多因子合成框架。

**前置**：V2.10 截面因子评估

**为什么基本面提前到 V2.11**：原计划基本面在优化器之后，但实际上：
1. A 股量化私募 80%+ 使用基本面因子（PE/PB/ROE），纯量价因子同质化严重
2. AlphaCombiner 没有基本面因子就只能组合技术因子，价值有限
3. 基本面因子是后续 ML Alpha 的核心特征输入

**扩展模块**：ez/data/providers/ (fundamental), ez/factor/builtin/ (fundamental), ez/portfolio/ (alpha_combiner)

### 交付

**基本面数据 (D1-D2)**:
- [ ] D1. FundamentalDataProvider — Tushare 财务数据接入
  - income（利润表）: revenue, net_profit, gross_margin
  - balancesheet（资产负债表）: total_assets, total_equity, debt_ratio
  - cashflow（现金流量表）: operating_cf, investing_cf, financing_cf
  - daily_basic: pe_ttm, pb, ps_ttm, dv_ratio, total_mv, circ_mv
  - **Point-in-time 对齐**：用 ann_date（公告日）而非 end_date（报告期），防前瞻偏差

  - **Growth**: revenue_growth_yoy, profit_growth_yoy, roe_change
  - **Size**: ln(market_cap), ln(circ_mv)
  - **Liquidity**: turnover_rate, amihud_illiquidity
  - **Leverage**: debt_ratio, interest_coverage

**行业因子 (D3)**:

  - 行业虚拟变量（one-hot），用于中性化和行业约束
  - 行业动量因子：过去 N 日行业指数收益排名
  - 行业内相对强弱：个股收益 - 所属行业均值

**Alpha 组合 (F3)**:
- [ ] F3. AlphaCombiner — 多因子合成框架

  - ICIR 加权：按 IC / std(IC) 分配（奖励稳定的因子）
  - 因子正交化：Gram-Schmidt 去除因子间冗余
  - 合成信号 = Σ(weight_i × zscore(factor_i))

**组合参数搜索 (NEW)**:
- [ ] U3. PortfolioParameterSearch — 组合策略参数网格/随机搜索
  - 复用 CandidateSearch 模式，适配组合策略
  - 参数范围定义 → 批量运行 → 按夏普/收益排名
  - 支持 TopNRotation (top_n × factor) 和内置策略参数

**数据质量 (NEW)**:
- [ ] U4. DataQualityDashboard — 数据完整性检查
  - 各标的缺失天数统计
  - 价格异常检测（跳空 > 30%、零价格、负价格）
  - 最后更新时间

## V2.12 — 组合优化 + 归因 + 风控

**目标**：从"等权持仓"升级到"约束优化 + 指数增强 + 绩效归因 + 风控"。

**前置**：V2.11 基本面因子 + AlphaCombiner

**扩展模块**：ez/portfolio/ (optimizer, benchmark, attribution, risk_manager)

### 交付

**组合优化 (F4)**:
- [ ] F4. PortfolioOptimizer — long-only 约束优化
  - 均值-方差 (Markowitz)：max(return - λ * variance)，s.t. w >= 0, Σw = 1
  - 最大分散化：max(Σ w_i σ_i / σ_portfolio)
  - 风险平价：使每只股票对组合风险的贡献相等
  - 约束：单股最大权重（默认 5%）、行业最大权重（默认 30%）、最大换手率
  - **所有优化器强制 w >= 0（long-only）**

**指数增强 (F5)**:
- [ ] F5. IndexEnhancement — 基准跟踪 + 主动偏离
  - 基准定义：CSI300 / CSI500 / CSI1000 成分股 + 权重（Tushare `index_weight`）
  - 主动权重 = 优化目标权重 - 基准权重
  - 跟踪误差约束：TE = std(R_portfolio - R_benchmark) <= 阈值（默认 5%）

**归因分析 (F6)**:
- [ ] F6. Attribution — 收益分解
  - Brinson 归因：配置效应 + 选股效应 + 交互效应
  - 因子归因：各因子对组合收益的贡献 = Σ(factor_exposure_i × factor_return_i)
  - 交易成本归因：总收益 - 毛收益 = 交易成本影响

**风控层 (D4)**:
- [ ] D4. RiskManager — 组合级风控引擎
  - 单股最大仓位限制（默认 5%）、行业最大仓位限制（默认 30%）
  - 最大回撤熔断：回撤 > 阈值 → 强制减仓（默认 20%）
  - 换手率上限（默认单边 30%）
  - 风控检查嵌入 PortfolioEngine.rebalance()

**报告导出 (NEW)**:
- [ ] U5. ReportExport — 策略研究报告生成
  - PDF/HTML 报告：净值曲线 + 指标表 + 归因饼图 + 交易统计
  - 团队分享友好

**前端**:
- [ ] 组合构建 sub-tab：Alpha 组合方式 + 优化目标 + 约束参数 + 指数增强开关
- [ ] 归因分析 sub-tab：Brinson 配置/选股归因图 + 因子归因 + 交易成本归因
- [ ] 风控参数配置 + 风控事件日志

### Exit Gate

- [ ] 优化器输出权重：全部 >= 0、和 = 1.0、单股 <= 约束上限
- [ ] Brinson 归因：配置 + 选股 + 交互 ≈ 总超额（误差 < 1bp）
- [ ] 风控熔断：注入 20% 回撤场景 → 减仓触发 → 会计不变量仍成立
- [ ] CSI300 指数增强可运行

## V2.13 — ML Alpha + 多策略组合

**目标**：机器学习因子框架（防前瞻 walk-forward）+ 多策略集成。

**前置**：V2.12 优化器 + 归因 + 风控

> **为什么 ML 需要框架级支持**：
> 现有 Strategy.generate_signals() 接收全量历史数据。如果用户在其中训练模型，
> 模型会看到未来数据（前瞻偏差），回测结果必然虚高。框架必须强制 walk-forward
> 训练/预测分离，用户无法绕过。

**扩展模块**：ez/portfolio/ (ml_alpha, ml_diagnostics, ensemble)

### 交付

**ML Alpha (F8-F9)**:
- [ ] F8. MLAlpha — Walk-forward ML 因子框架
  - **用户提供**：model_factory（返回 sklearn-compatible 模型）、features、target
  - **框架保证**：walk-forward 时间切割，用户无法触及训练数据划分
  - 参数：train_window（504 日 ≈ 2 年）、retrain_freq（21 日 ≈ 月度）、purge_days（5 天防泄漏）、embargo_days
  - **防泄漏**：purge + embargo 保证训练标签与预测日不重叠
  - 兼容 sklearn API：LinearRegression, Ridge, RandomForest, LightGBM, XGBoost
  - 输出：CrossSectionalFactor 格式，直接喂给 AlphaCombiner / PortfolioStrategy
- [ ] F9. MLDiagnostics — 过拟合检测
  - 特征重要性追踪（不稳定 = 过拟合）
  - IS/OOS IC 衰减（衰减 > 50% = 严重过拟合）
  - 预测自相关 + 换手分析

**多策略组合 (D5)**:
- [ ] D5. StrategyEnsemble — 多策略长仓组合
  - 组合方式：等权、风险预算、IC 加权
  - 去相关：策略间收益相关矩阵，降权高相关策略

**代码编辑器 + Agent (F7)**:
- [ ] F7. CodeEditor + Agent 支持 ML
  - "新建 ML Alpha" 模板 + `ml_alphas/` 目录
  - sandbox 白名单：sklearn, lightgbm, xgboost
  - Agent 工具：`create_ml_alpha`, `evaluate_cross_factor`, `run_portfolio_optimization`

**前端**:
- [ ] 因子研究 sub-tab 增加 ML Alpha 配置面板
- [ ] ML 诊断面板（特征重要性 + IS/OOS IC + 换手）
- [ ] 多策略组合页：选择子策略 + 组合方式 + 对比图

### Exit Gate

- [ ] MLAlpha walk-forward：策略函数只收到 T-1 及之前的特征（purge+embargo 验证）
- [ ] ML 特征快照和模型快照可哈希、可重放
- [ ] Agent 能自主完成一轮 ML alpha 探索
- [ ] 多策略组合器输出权重合规（>= 0, 和 <= 1.0）

## A 股约束备忘

> **贯穿所有版本的硬性约束**：
> - **不能做空个股**：A 股散户/多数私募无法融券做空。所有策略信号 ∈ [0, 1]，不支持 [-1, 1]。
> - **T+1**：买入当天不能卖出（V2.6 已实现）。
> - **涨跌停**：10%/20% 日涨跌幅限制，涨停不可买、跌停不可卖（V2.6 已实现）。
> - **整手**：最小交易单位 100 股（V2.6 已实现）。
> - **Long-only 约束**：所有组合优化器必须强制 w >= 0。
> - **不做配对交易空腿**：配对交易需做空一腿，A 股不可行。
> - **市场中性需股指期货**：做空对冲需要股指期货（IF/IC/IM），属于跨市场，推迟到 V3.x 有实盘基础设施后考虑。
> - **印花税**：卖出时 0.05% 印花税，需在交易成本中计入。

## 优先级较低（不是不做，时机未到）

- 高频事件引擎（日线级事件总线优先，高频后续可扩展）
- 分钟/Tick 数据（日线足以覆盖 A 股主流策略，分钟级按需扩展）
- 做空/市场中性（需股指期货数据+跨市场引擎，V3.x 考虑）
- 配对交易（A 股不可做空，跨品种套利属于期货市场）
- Rust Live 核心（Python Paper OMS 先跑通，有瓶颈再迁移）
- 美股/港股深度支持（A 股优先）
- 多用户协作（RBAC + workspace 隔离，团队规模扩大后考虑）
- 期权/衍生品（需要专门的定价引擎和数据源）
- 策略回测可视化增强（逐笔交易在 K 线图上标注）
- 深度学习模型（Transformer/LSTM — MLAlpha 框架支持 sklearn API，DL 需额外适配器）
