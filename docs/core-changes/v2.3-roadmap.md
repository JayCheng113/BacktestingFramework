## V2.3 — Correctness Hardening (正确性封顶)

**目标**：把现有研究链路做成"可证明正确"。

**模块**：ez/backtest, ez/core/ts_ops.py, ez/core/_cpp/ts_ops.cpp, tests/

### 交付

- [x] A1. 会计不变量测试：逐 bar `abs(cash + position_value - equity[i]) <= eps` — 173 tests (含 equity-from-returns 独立 oracle)
- [x] A2. C++/Python 双路径对拍框架：同输入 _USE_CPP=True/False，逐列比对 — 109 tests (含 span=1、大数值 1e12)
- [x] A3. rolling_std O(n) Welford 优化 — 5.2x vs Python (周期性重算防大数漂移)
- [x] A4. 架构门禁测试 (tests/test_architecture/) — 47 tests
- [x] A5. ewm_mean(span=1) NaN bug 修复 — Codex review 发现并修复

### 清理

- [x] 统一参数验证入口到 ts_ops.py（Python 和 C++ 双层校验已有，确认无遗漏）
- [x] eps 容差分级：EPS_FUND=0.01(资金), EPS_RATE=1e-10(收益率), EPS_STD=1e-6(标准差)

### Exit Gate

- [x] 全量测试通过 — 669 pass, 10 skip, 1 DuckDB 锁(已知)
- [x] C++/Python 对拍差异 < 分级 eps — 109 tests 全部通过
- [x] rolling_std C++ 性能优于 Python — 5.2x
- [x] 性能基线不回退（benchmark.py 结果 >= V2.2）— 60K+ bars/sec
- [x] 0 新增 Critical/Important bug (ewm_mean span=1 bug 已在本版修复)

### Web 策略/因子代码编辑器 (V2.5 子项)

**目标**：人类研究员在 Web 端在线创建策略/因子，不需要本地开发环境。

| 编号 | 交付 | 说明 |
|------|------|------|
| C1 | 模板初始化 | 选择 Strategy/Factor 类型 → 生成 Python 模板代码（继承 ABC，填充示例逻辑） |
| C2 | 代码编辑器 | 内嵌 Monaco Editor（VS Code 同款），Python 语法高亮 + 自动补全 |
| C3 | 自动 Contract Test | 保存时后端执行 contract test，返回通过/失败 + 错误详情 |
| C4 | 注册 & 运行 | 通过 contract test 后自动注册到 Strategy/Factor registry，可在回测/实验中使用 |

**API**:
- `POST /api/strategies/create` — 提交代码 → contract test → 注册
- `GET /api/strategies/{name}/source` — 获取策略源码
- `PUT /api/strategies/{name}/source` — 更新策略源码

**安全**: 用户代码在沙箱中执行（受限 import，超时控制）。详细沙箱设计在实现时确定。

## V2.7 — Web AI Assistant (Web 端 AI 对话助手)

**目标**：在 Web 端内嵌 AI 对话框，辅助人类研究员进行策略开发。

**新模块**：web/components/AIChat, ez/api/routes/chat.py

### 交付

- [ ] D1. 对话 UI — Web 端聊天面板（输入框 + 消息流 + 代码高亮）
- [ ] D2. 策略实现助手 — 用户描述想法，Agent 生成策略代码并填入编辑器
- [ ] D3. Debug 助手 — 用户粘贴错误/结果，Agent 分析原因并给出修复建议
- [ ] D4. 结果解释 — 选中某次实验，Agent 解释 metrics/gate 结果含义
- [ ] D5. 上下文感知 — Agent 能读取当前编辑器代码、实验结果、因子分析数据

### 设计要点

- 后端: 流式 SSE 响应（`/api/chat`），接入 Claude API 或本地模型
- 前端: 可折叠侧边栏，支持 Markdown + 代码块渲染
- 权限: Agent 只能读取数据和生成代码，不能直接修改核心文件或下单
- 上下文: 通过 system prompt 注入当前 Strategy 源码 + 最近实验结果

### Exit Gate

- [ ] 对话流畅，能正确生成 Strategy/Factor 代码
- [ ] Debug 建议与实际错误一致
- [ ] Agent 无法触发超出研究范围的操作
