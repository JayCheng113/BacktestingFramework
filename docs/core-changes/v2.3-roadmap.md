# V2.3+ Roadmap

## Principle

> "先做正确性封顶，再做 Agent 闭环" — Codex + 内部共识

Agent-Native 不是"AI 帮写代码"，而是：
- 机器可读契约（策略/因子/数据接口明确且稳定）
- 自动验证与自动反馈（不是人工逐个 review）
- 可编排、可追踪、可恢复（失败可重试，结果可审计）
- 人类只做高层约束与审批，不做每一步操作员

---

## V2.3 — Correctness Hardening (正确性封顶)

### A1. Accounting Invariant Test (per-bar reconciliation)
每一 bar 验证：`abs(cash + position_value - equity_curve[i]) <= eps` (eps=0.01，1分钱精度)
全局：`abs(initial_capital + sum(realized_pnl) - sum(fees) + unrealized_pnl - final_equity) <= eps`
注：浮点运算不能用严格 ==，统一用 eps 容差。

### A2. Python/C++ Dual-Path Comparison Framework
同输入同参数，C++ on/off 两条路径跑完，逐列数值对比。
覆盖：ts_ops 5 函数 × (clean data + NaN data + edge cases)

### A3. rolling_std Welford Online Algorithm
当前 O(n*window) ~1x vs pandas，优化为 O(n)。

## V2.4 — Agent Loop (Agent 闭环)

> 目标：Agent 提交实验 → 系统自动评估 → 结构化结果 → Agent 决策下一步

### B1. RunSpec — 标准化实验输入
`ez/agent/run_spec.py`: 定义 RunSpec dataclass
- 数据范围（symbol, market, period, date range）
- 策略标识 + 参数
- 交易成本参数（commission, slippage）
- 验证配置（n_splits, train_ratio）
- 随机种子（可复现）

### B2. Runner — 组合执行器
`ez/agent/runner.py`: 调用已有模块执行完整流程
- 因子评估（FactorEvaluator）
- 单次回测（VectorizedBacktestEngine）
- Walk-Forward 验证（WalkForwardValidator）
- 显著性检验（compute_significance）
- 返回结构化 RunResult

### B3. Agent Gate — 自动晋级规则
`ez/agent/gates.py`: 可配置的策略准入门禁
- `sharpe_ci_lower > 0`（统计显著性）
- `p_value < 0.05`（Monte Carlo 显著）
- `is_vs_oos_degradation < 阈值`（防过拟合）
- `max_drawdown > 下限`（风险控制）
- Gate 输出：PASS / FAIL + 原因

### B4. Report — 结构化输出
`ez/agent/report.py`: Agent 可解析的结果格式
- JSON/dict 输出，便于 Agent 下一轮决策
- 包含：指标、显著性、WFO 结果、Gate 判定

### B5. Experiment Persistence — 实验台账
修改 `ez/data/store.py`: 新增 runs / run_artifacts 表
- 记录：输入 spec、代码版本、指标、是否通过 gate
- 支持 Agent 查询历史实验、比较结果

### B6. Experiments API
`ez/api/routes/experiments.py`:
- POST /experiments/submit — 提交 RunSpec
- GET /experiments/{id} — 查询状态/结果
- GET /experiments — 列表（可过滤/排序）
- POST /experiments/{id}/promote — 标记晋级

## V2.5 — Scale (规模化)

- 多标的回测 + 截面策略
- 策略元数据（ID / version / tags / risk_class）
- 回测可复现（数据快照哈希 + 依赖版本入库）
- 多重检验控制：FDR/Bonferroni 或外层留出集（与参数搜索同步引入，防数据挖掘偏差）
- 更多数据源
- MarketRules 模块（T+1、涨跌停、整手、停牌）

## V3 — Pre-Production (实盘前置)

- Paper trading broker（仿真实盘）
- 最小事件总线（bar/order/fill 事件驱动 OMS）
- PreTradeRiskEngine
- 风险限额 + 审批流
- 审计日志
- 小规模真实资金

## 优先级较低（不是不做，时机未到）
- 高频事件引擎: 当前无 tick 数据，日线级事件总线优先，高频后续可扩展
- engine._simulate C++ 下沉: 60K bars/sec 日线够用，暂不需要
- Monte Carlo C++: 1000 permutations 10ms 不值得
- CRTP/Eigen 重度 C++: 薄封装够用
