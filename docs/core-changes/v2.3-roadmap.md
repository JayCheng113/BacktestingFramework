### Web 策略/因子代码编辑器 (V2.5 子项)

**目标**：人类研究员在 Web 端在线创建策略/因子，不需要本地开发环境。

| 编号 | 交付 | 说明 |
|------|------|------|
| C1 | 模板初始化 | 选择 Strategy/Factor 类型 → 生成 Python 模板代码（继承 ABC，填充示例逻辑） |
| C2 | 代码编辑器 | 内嵌 Monaco Editor（VS Code 同款），Python 语法高亮 + 自动补全 |
| C3 | 自动 Contract Test | 保存时后端执行 contract test，返回通过/失败 + 错误详情 |
| C4 | 注册 & 运行 | 通过 contract test 后自动注册到 Strategy/Factor registry，可在回测/实验中使用 |

**API**:
- `POST /api/strategies/create` — 提交代码 → contract test → 注册
- `GET /api/strategies/{name}/source` — 获取策略源码
- `PUT /api/strategies/{name}/source` — 更新策略源码

**安全**: 用户代码在沙箱中执行（受限 import，超时控制）。详细沙箱设计在实现时确定。

## V2.7 — LLM 集成 + Web Coding Assistant (人类驱动的 AI 助手)

**目标**：在 Web 端内嵌类似 Claude Code 的 AI 助手，辅助人类研究员更快地写策略、调试、理解结果。**人类驾驶，Agent 副驾**。

**新模块**：ez/llm/ (LLM 抽象层), ez/agent/tools.py, ez/api/routes/chat.py, web/components/AIChat

> **与 V2.8 Research Agent 的本质区别**：
> - Web Assistant (V2.7) = 人类说"帮我写个 RSI 策略"，Agent 写代码、跑测试、人类确认
> - Research Agent (V2.8) = 人类说"探索 A 股动量策略空间"，Agent 自主生成 N 个候选、批量实验、筛选汇报

### LLM Provider 抽象层 (`ez/llm/`)

支持多家 LLM，**中国国内可用为首要考虑**。

| Provider | 国内可用 | tool_use/function_calling | 优先级 |
|----------|---------|--------------------------|--------|
| **DeepSeek** | ✅ 直连 | ✅ OpenAI 兼容格式 | **P0** |
| **Qwen (通义千问)** | ✅ 直连 | ✅ 原生 tool_use | P1 |
| Claude (Anthropic) | ❌ 需代理 | ✅ tool_use | P2 |
| OpenAI | ❌ 需代理 | ✅ function_calling | P2 |
| 本地模型 (Ollama/vLLM) | ✅ 完全离线 | ⚠️ 部分支持 | P1 |

```python
# ez/llm/provider.py
class LLMProvider(ABC):
    def chat(self, messages, tools=None) -> LLMResponse: ...
    def stream_chat(self, messages, tools=None) -> Iterator[LLMEvent]: ...

class DeepSeekProvider(LLMProvider): ...   # 默认
class QwenProvider(LLMProvider): ...
class AnthropicProvider(LLMProvider): ...
class OpenAICompatibleProvider(LLMProvider): ...  # 覆盖 Ollama/vLLM
```

```yaml
# config.yaml
llm:
  provider: deepseek           # deepseek | qwen | anthropic | openai | local
  api_key: ${DEEPSEEK_API_KEY}
  model: deepseek-chat
  base_url: ~                  # 留空用默认，本地模型填 http://localhost:11434
```

### 典型使用场景

```
用户: "帮我写一个基于 RSI 超卖的反转策略，RSI < 30 买入，> 70 卖出"

Agent 行动 (用户可见):
  → [读取] ez/strategy/base.py 接口定义
  → [读取] ez/factor/builtin/technical.py 中 RSI 因子

Agent: "策略已创建并通过测试。回测结果: Sharpe 0.8, 最大回撤 15%。
        建议：RSI 阈值可以参数化，方便后续优化。要我改吗？"
```

## V2.8 — Autonomous Research Agent (自主研究 Agent 流水线)

**目标**：Agent 自主探索策略空间 — 生成假设、写代码、批量实验、评估筛选、迭代优化。**Agent 驾驶，人类审核**。

**扩展模块**：ez/agent/ (hypothesis.py, code_gen.py, loop_controller.py)

> **与 V2.7 Web Assistant 的本质区别**：
> - V2.7 = 同步、交互式、每步人类确认
> - V2.8 = 异步、自主式、人类只审核最终批次结果

### 研究 Agent 流水线

```
人类设定目标
  "探索 A 股日线动量类策略，Sharpe > 1，回撤 < 20%"
     │
     ▼
┌─────────────────────────────────────────┐
│  Hypothesis Generator (LLM)             │
│  输入: 目标 + 已有策略 + 市场数据概况    │
│  输出: N 个策略假设 (自然语言描述)       │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Code Generator (LLM)                    │
│  输入: 假设 + Strategy ABC 接口          │
│  输出: N 个 .py 策略文件                 │
│  → 自动 contract test → 过滤编译失败     │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Batch Runner (已有 V2.4 Runner)         │
│  对每个策略执行: backtest + WFO + sig    │
│  → ExperimentStore 持久化                │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Research Gate (已有 V2.4 Gate)           │
│  过滤: Sharpe/DD/trades/significance     │
│  → 淘汰不达标策略                        │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Analyzer (LLM)                          │
│  输入: 通过/失败的策略 + metrics          │
│  输出: 改进建议 / 新假设方向              │
└──────────────┬──────────────────────────┘
               ▼
┌─────────────────────────────────────────┐
│  Loop Controller                         │
│  决策: 继续迭代? 尝试新方向? 停止?       │
│  条件: 达到目标 / 超出预算 / N轮无进展   │
└──────────────┬──────────────────────────┘
               ▼
        研究报告 → 人类审核
```

### 与 V2.5 Scale 的区别

| 维度 | V2.5 Scale | V2.8 Research Agent |
|------|-----------|-------------------|
| 搜索方式 | 参数网格/随机搜索（固定策略结构） | LLM 生成新策略结构 |
| 创造性 | 无（只调参数） | 有（生成全新逻辑） |
| 人类参与 | 设定搜索空间 | 设定研究目标 |
| 输出 | 最优参数组合 | 新策略代码 + 研究报告 |

### 交付

- [ ] E1. Hypothesis Generator — LLM 生成策略假设
- [ ] E2. Code Generator — LLM 写策略代码 + 自动 contract test 过滤
- [ ] E3. Batch Executor — 批量调度 Runner，并发执行
- [ ] E4. Analyzer — LLM 分析实验结果，提出改进方向
- [ ] E5. Loop Controller — 循环控制（预算、轮次、收敛判定）
